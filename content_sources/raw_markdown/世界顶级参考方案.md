## 二、世界顶级的核心特征

通过分析以上产品，总结出顶级关系训练系统的8个共性特征：

### 特征1：情绪量化体系
- **Gottman**用"情绪温度计"（0-100）追踪关系健康度
- **Lasting**有每日"关系温度"测验
- **本项目**：已有70个情绪词+10级强度，完全可对标

### 特征2：对比驱动学习（Difference-Driven Learning）
- **所有顶级产品都内嵌"你的回答 vs 理想回答"机制**
- Relish/Lasting用视频示范，本项目用文字对比引擎
- 这是复训率最高的功能模块

### 特征3：间隔重复（Spaced Repetition）
- **Anki原理**：遗忘曲线复习
- 关系技能本质是条件反射，需要重复暴露
- Lasting的"每周复习"、Paired的"回顾上周答案"都是SR实现

### 特征4：游戏化（Gamification）
- **Paired**的问答游戏日活留存率极高
- 等级系统、连胜系统、成就徽章
- 本项目已有"八阶路径"，需强化游戏化细节

### 特征5：依恋类型诊断
- **成人依恋访谈（AAI）数字化版本**
- 所有顶级产品都有依恋测试作为入口
- 决定后续内容推送的个性化逻辑

### 特征6：每日微训练（Micro-Training）
- **Lasting**：每天5分钟
- **Paired**：每天一个问答，3分钟
- 核心洞察：用户注意力窗口极短，必须"轻量化、高频次"

### 特征7：可汗学院式进展追踪
- 数据仪表盘：雷达图、进度条、历史曲线
- 让用户看到自己的成长轨迹
- 本项目Dashboard已有雏形，需增强数据可视化

### 特征8：真人元素（Human Touch）
- **Relish**最强差异化：真人教练介入
- **Gottman**权威背书
- 数字系统无法完全替代真人关系顾问

## 四、循序渐进路线图

### 第一阶段：打磨核心体验（MVP完善）预计2-4周

**目标**：让现有功能真正可玩、打磨核心差异

#### 1.1 情绪量化体系
```
新增：
- "关系温度计"：每日情感状态打卡（0-100滑块）
- 每周生成"情感周报"（极简图表）
- 追踪：关系满意度、冲突频率、亲密度三维度
```

#### 1.2 每日微训练节拍
```
新增：
- 每日推送：1个情绪识别 + 1个对比回应（≤5分钟）
- Push通知提醒（可选）
- 完成后才能解锁第二天的"故事续更"
- 断签惩罚：连胜中断显示红色警告
```

#### 1.3 对比引擎升级
```
新增：
- 三版本对比：柔和版 / 张力版 / 幽默版（每条样本都要有）
- "你的得分"：从0-100评分，给出差距原因
- 错题本：自动收录所有答错样本，按艾宾浩斯曲线推送
```

#### 1.4 依恋测试强化
```
新增：
- 36题成人依恋测试（改编自ECR-R）
- 入口改为：先测依恋类型 → 再推荐八阶起点
- 依恋类型决定：样本推送优先级、提示风格
```

### 第二阶段：AI伴侣 & 情感识别 预计4-8周

**目标**：用AI实现"24小时陪练"

#### 2.1 AI对话模拟器
```
技术方案：
- 接入DeepSeek API（你已有的配置）
- 场景：约会初期/暧昧期/冲突期/平淡期
- AI扮演不同依恋类型（焦虑型/回避型/安全型）
- 实时评分+提示：下一句该怎么说

参考：Bliss、Relate的对话分析思路
GitHub参考：aayushdhanotia12/WeHeal（AI心理对话）
```

#### 2.2 语音情绪识别（可选进阶）
```
技术方案：
- 接入语音转文字（Web Speech API）
- 接入情绪分析（ HuggingFace transformers）
- 场景：用户语音输入→情绪标签+建议回复
- 简化版：纯文字情感分析

GitHub参考：declare-lab/MELD（对话情绪识别⭐1047）
GitHub参考：Speech-Emotion-Recognition（语音情绪识别）
```

#### 2.3 对话记录分析
```
功能：
- 用户粘贴聊天记录截图/文字
- AI分析：情绪流动、潜在冲突点、改进建议
- 类似"Relate的Chat Analysis"功能
```

### 第三阶段：社交化 & 商业化 预计8-16周

**目标**：从单人工具→关系网络

#### 3.1 双人互动系统
```
新增：
- 邀请伴侣加入（验证码机制）
- 双人问答游戏（Paired模式）
- 双方进度对比仪表盘
- "关系挑战赛"：7天同步任务
```

#### 3.2 社区互助
```
新增：
- 匿名关系问答（类知乎）
- 成功案例展示（用户自愿分享）
- 小组机制：同依恋类型互助群
```

#### 3.3 真人教练对接
```
新增：
- 教练入驻系统
- 付费1对1预约
- 教练端App（查看用户数据）
- 这是Relish的核心壁垒
```

#### 3.4 高级付费会员
```
Freemium模型：
- 免费：每日1次训练、基础样本库
- 付费($9.9/月)：无限训练、AI伴侣、视频示范、情感报告
- 付费($29.9/月)：双人模式、真人教练预约优先
```

### 第四阶段：内容生态 & 扩展 预计16周+

**目标**：从工具→平台→生态

#### 4.1 用户生成内容（UGC）
```
- 场景样本开放提交
- 审核后加入样本库（标注贡献者）
- 建立"场景市场"：用户可购买稀缺场景
```

#### 4.2 细分赛道扩展
```
现有：两性关系（情侣）
扩展方向：
- 亲子关系（父母子女）
- 职场关系（同事/上下级）
- 友情（朋友沟通）
- 自我关系（与自己对话）
```

#### 4.3 硬件/可穿戴
```
- Apple Watch情感追踪
- 与Oura Ring/Circular数据打通
- 实现"被动情绪监测"
```

## 五、技术升级路线

### 当前栈 vs 目标栈

```
当前：                              目标：
Vue3 + FastAPI                  Vue3 + FastAPI (保留)
SQLite (本地)              →      PostgreSQL + Redis (生产)
纯前端存储                 →      用户账户系统 + 云同步
情绪词典（静态JSON）        →      动态情绪知识图谱
对比引擎（规则匹配）        →      LLM驱动的智能对比
固定样本库                 →      LLM生成 + UGC扩充
单用户                     →      多用户 + 伴侣配对
中文                       →      中英双语（可配置）
```

### AI能力分层

```
Level 1 (当前)：规则引擎对比
  - 原回应 vs 理想回应（字符串匹配）
  - 固定样本库

Level 2 (下一版本)：LLM增强
  - DeepSeek API生成多版本对比
  - LLM评估用户回答并打分
  - 智能错题本（LLM生成解释）

Level 3 (中版本)：AI伴侣
  - 多轮对话模拟
  - 实时情绪检测
  - 自适应学习路径

Level 4 (远期)：多模态
  - 语音情绪分析
  - 视频微表情识别
  - 聊天记录智能分析
```
