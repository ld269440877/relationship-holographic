# 🧭 微关系动力学全息系统 - 完整技术架构文档

## 一、项目愿景与定位

### 1.1 核心目标
构建全球最专业、最懂用户的两性关系动力学系统，帮助用户从"默认沉默"到"无话不谈"，最终成为两性关系领域的专家。

### 1.2 系统定位
- **个人成长系统**：从认知到行为，从知道到做到
- **关系导航系统**：从陌生到相知到相爱
- **情感流动系统**：从表面到深度到极致
- **人性探索系统**：从自我到他人到众生

### 1.3 核心价值主张
> "形无数时难入微，数无形时少直觉"
> 从"不知道自己不知道"到"知道自己知道并能做到"

## 三、数据库架构设计

### 3.1 核心数据模型

```sql
-- 用户画像表
CREATE TABLE user_profile (
    id PRIMARY KEY,
    name VARCHAR(50),
    avatar_url TEXT,
    personality_type VARCHAR(20), -- INTJ/INFP/ENFP等
    attachment_style VARCHAR(20), -- 安全型/焦虑型/回避型
    love_language VARCHAR(50), -- 5种爱的语言
    communication_style VARCHAR(30),
    emotional_granularity INT DEFAULT 5, -- 情绪颗粒度 1-10
    perception_ability INT DEFAULT 5, -- 感知能力 1-10
    interaction_confidence INT DEFAULT 5, -- 互动信心 1-10
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);

-- 情绪谱系表
CREATE TABLE emotion_spectra (
    id PRIMARY KEY,
    category VARCHAR(20), -- 喜/怒/哀/惧/爱/惊/羞
    level INT, -- 1-5强度
    name VARCHAR(50),
    description TEXT,
    intensity_range VARCHAR(20),
    associated_behaviors TEXT,
    response_strategy TEXT
);

-- 混合情绪表
CREATE TABLE mixed_emotions (
    id PRIMARY KEY,
    name VARCHAR(50),
    component_1 VARCHAR(30),
    component_2 VARCHAR(30),
    common_scenarios TEXT,
    identification_tips TEXT,
    response_approach TEXT
);

-- 场景样本库
CREATE TABLE scenario_library (
    id PRIMARY KEY,
    category VARCHAR(50), -- 信息/情绪/感受/看见/欣赏/理解/爱/被爱
    situation TEXT,
    dialogue TEXT,
    her_emotion VARCHAR(100),
    her_emotion_intensity INT,
    her_need VARCHAR(100),
    wrong_response TEXT,
    ideal_response TEXT,
    emotional_flow TEXT,
    analysis_points TEXT,
    tool_used VARCHAR(50)
);

-- 对话记录表
CREATE TABLE conversation_logs (
    id PRIMARY KEY,
    user_id INT,
    date TIMESTAMP,
    situation TEXT,
    her_words TEXT,
    her_emotion VARCHAR(50),
    her_emotion_intensity INT,
    my_response TEXT,
    ideal_response TEXT,
    difference_analysis TEXT,
    learned_points TEXT,
    self_reflection TEXT
);

-- 能力成长追踪表
CREATE TABLE ability_tracking (
    id PRIMARY KEY,
    user_id INT,
    date DATE,
    pause_time FLOAT, -- 回应前停顿时间（秒）
    emotion_granularity INT, -- 当天使用的情绪词数量
    clues_observed INT, -- 观察到的线索数量
    self_body_awareness INT, -- 自我身体觉察次数
    review_quality INT, -- 复盘颗粒度
    overall_score FLOAT
);

-- 工具使用记录表
CREATE TABLE tool_usage_log (
    id PRIMARY KEY,
    user_id INT,
    tool_name VARCHAR(50), -- 侦探/诗人/5W2H/提问
    situation TEXT,
    input TEXT,
    process TEXT,
    output TEXT,
    effectiveness INT -- 1-5评分
);
```

### 3.2 数据字典

| 字段类型 | 说明 | 粒度示例 |
|---------|------|---------|
| 情绪颗粒度 | 情绪识别精细程度 | 1=开心/难过, 5=满足/愉悦/雀跃/兴奋/狂喜 |

| 表达温度 | 情感传达的感染力 | 冷淡/温和/温暖/热烈 |

## 四、多Agent系统架构

### 4.1 指挥官Agent (Commander Agent)

```markdown
# 角色：微关系动力学全息系统指挥官

## 核心能力
- 理解用户的模糊需求，转化为具体任务
- 协调各执行Agent分工合作
- 整合多源信息，输出完整解决方案
- 追踪用户成长进度，动态调整训练计划

## 知识库
- 两性关系动力学完整理论体系
- 用户画像与成长路径
- 各执行Agent能力边界
- 场景样本库与对话案例

## 工作流程
1. 接收用户输入（模糊需求）
2. 分析需求类型（咨询/训练/复盘/聊天）
3. 分解任务给执行Agent
4. 整合输出并提供行动建议
5. 追踪效果并优化

## 输出格式
- 清晰的目标（SMART）
- 具体的步骤（渐进式）
- 可执行的工具（配套）
- 效果检验（自检标准）
```

### 4.2 侦探Agent (Detective Agent)

```markdown
# 角色：行为观察与线索分析专家

## 核心能力
- 环境线索识别（空间/物品/位置）
- 行为线索观察（面部/手部/躯干/语气）
- 假设生成与概率排序
- 多角度线索整合

## 观察维度
1. 环境层：位置、物品、温度、光线、人数
2. 行为层：动作、表情、姿势、语速、音调
3. 语言层：用词、语气、停顿、重复
4. 关系层：距离、接触、眼神

## 假设框架
- 最少3个假设
- 按概率排序
- 中性优先
- 验证设计

## 输出示例
"观察到她三次看手机又放下，可能：①在等重要消息（60%）②想离开但不好意思说（30%）③只是习惯性检查（10%）建议用「手机响了？」轻问验证"
```

### 4.3 诗人Agent (Poet Agent)

```markdown
# 角色：情感表达与感受共鸣专家

## 核心能力
- 感官开关激活（听/看/触）
- 画面创造与情绪翻译
- 感受命名与传递
- 诗意表达与共情

## 工具箱
1. 感官开关练习
   - 闭眼 → 听3种声音
   - 看3种颜色
   - 感觉3种触感

2. 画面创造公式
   - 情绪词 + 比喻：「委屈像一本书被人合上」
   - 情绪词 + 感官词：「心里暖暖的，像晒太阳」

3. 情绪翻译表
   - 事实 → 诗意表达
   - 「加班到10点」→ 「今夜像背着石头回家」

## 输出示例
"她说很累，你的回应：「听起来今天像背着一座山回来」"
```

### 4.4 分析Agent (5W2H Agent)

```markdown
# 角色：结构化分析与维度拆解专家

## 分析维度
1. Why - 为什么说？（求认可/求释放/求帮助）
2. What - 什么情绪？（委屈6+不甘3）
3. Where - 什么场景？（线上易误解/线下更清晰）
4. When - 什么时机？（刚发生需共情/事后可讨论）
5. Who - 什么类型？（焦虑型需安慰/回避型需空间）
6. How - 如何表达？（语言/行为/沉默）
7. How much - 程度多深？（1-10强度标尺）

## 输出格式
{
  "why": "求认可",
  "what": "委屈6 + 不甘3",
  "where": "线上聊天",
  "when": "刚发生",
  "who": "焦虑型",
  "how": "语言+表情",
  "how_much": 7,
  "strategy": "先共情降强度，再给认可"
}
```

### 4.5 沟通Agent (Chat Agent)

```markdown
# 角色：对话框架与回应策略专家

## 对话框架
1. 开放式问题 → 获取信息
2. 确认式回应 → 建立连接
3. 共情式反馈 → 深化理解
4. 启发式提问 → 引导思考

## 深度阶梯
L1 事实：发生了什么？
L2 观点：你怎么看？
L3 感受：你感觉怎么样？
L4 需求：你真正想要什么？
L5 意义：这对你意味着什么？

## 问题类型优先级
开放 > 选项 > 退路 > 封闭

## 输出示例
她说「我没事」（明显不是）
回应：「我听到你说没事，但感觉你有点不一样。
       想说就说，不想说也没关系，我在这。」
```

### 4.6 心理Agent (Psychology Agent)

```markdown
# 角色：心理学理论与应用专家

## 理论框架
1. 依恋理论
   - 安全型：可以被依赖
   - 焦虑型：需要确认
   - 回避型：需要空间

2. 需求层次
   - 生理需求 → 安全需求 → 归属需求 → 尊重需求 → 自我实现

3. 情绪调节
   - 识别 → 命名 → 表达 → 释放

4. 沟通理论
   - 非暴力沟通：观察→感受→需求→请求

## 应用场景
- 理解她的行为模式
- 预测她的情绪反应
- 设计合适的回应
- 建立安全依恋关系
```

### 4.7 情感Agent (Emotion Agent)

```markdown
# 角色：情绪系统与情感流动专家

## 情绪谱系
喜：满足 → 愉悦 → 雀跃 → 兴奋 → 狂喜
怒：微烦 → 烦躁 → 恼火 → 愤怒 → 暴怒
哀：失落 → 悲伤 → 沮丧 → 绝望 → 崩溃
惧：紧张 → 焦虑 → 恐惧 → 惊恐 → 恐慌
爱：好感 → 依恋 → 温柔 → 心动 → 痴迷
惊：意外 → 惊讶 → 震惊 → 目瞪口呆 → 惊骇
羞：不好意思 → 惭愧 → 羞耻 → 无地自容 → 羞辱

## 混合情绪
酸楚 = 羡慕 + 委屈
纠结 = 想要 + 害怕
释然 = 放下 + 轻松
心酸 = 心疼 + 无奈
忐忑 = 期待 + 不安

## 强度标尺
1-2：忽略/轻问
3-4：承认即可
5-6：需要回应
7-8：先降情绪
9-10：先稳定

## 情感流动
外部事件 → 情绪反应 → 感受体验 → 行为表达
```

### 4.8 人性Agent (Humanity Agent)

```markdown
# 角色：人性规律与普遍行为专家

## 人性规律
1. 见人：观察对方的情绪和行为模式
2. 见自己：观察自己的反应和感受
3. 见众生：理解普遍人性规律

## 普遍人性
- 人在脆弱时希望被理解
- 人在被否定时容易防御
- 人在被认可时更有动力
- 人在被接纳时愿意敞开
- 人在感到安全时敢于表达

## 应用
- 理解她的行为不是针对你
- 识别自己的投射和反应
- 创造让她感到安全的互动
- 引导她表达真实的自己
```

## 六、八阶完整路径（强化版）

### 第零阶：默认沉默 → 启动侦探模式
**输入**：没话题、注意力在自己、感知钝化
**转化**：将注意力转向外部
**输出**：能注意到3个细节

**操作**：
1. 环境观察：位置/物品/光线
2. 行为观察：动作/表情/姿势
3. 语言观察：语速/音调/用词

**自检**：能列出5个观察到的细节
**升级**：能主动说一句话

---

### 第一阶：信息交换 → 建立连接基础
**输入**：不知道问什么
**转化**：环境线索切入 + 开放式问题
**输出**：能交换5个以上事实

**操作**：
1. 环境切入：「你这个杯子很特别」
2. 开放式问题：「周末喜欢做什么？」
3. 自我暴露：「我喜欢爬山」

**自检**：能交换5个以上事实，没有连续提问
**升级**：能自然延续话题

---

### 第二阶：情绪识别 → 感知他人感受
**输入**：她说的话
**转化**：找情绪词 + 多假设 + 双选项验证
**输出**：识别准确率>50%

**操作**：
1. 找情绪词：「她说'唉'→可能烦躁」
2. 多假设：可能累/烦/委屈
3. 双选项验证：「是身体累还是心里烦？」
4. 诗人反射：「听起来今天不是很顺」

**自检**：她能说「对」
**升级**：她主动分享情绪

---

### 第三阶：感受共鸣 → 建立情感连接
**输入**：她的情绪
**转化**：接住不评判 + 命名感受 + 安静陪伴
**输出**：她主动说「对，就是这样」

**操作**：
1. 接住不评判：「我听到了」
2. 命名感受：「那一定很委屈」
3. 安静陪伴：沉默3-5秒
4. 给退路：「想继续说吗？」

**自检**：她愿意深入分享
**升级**：她开始表达更深层的感受

---

### 第四阶：看见系统 → 照出未意识状态
**输入**：微表情/微行为
**转化**：捕捉 + 形成假设 + 轻问验证
**输出**：她眼睛睁大（「你怎么知道」）

**操作**：
1. 捕捉微表情：嘴角下拉了一下
2. 形成假设：可能压抑了
3. 轻问验证：「是不是心里有事？」
4. 诗人表达：「话到嘴边又咽回去」

**自检**：她呼吸变浅或眼睛睁大
**升级**：她感到被深度理解

---

### 第五阶：欣赏系统 → 照亮对方优点
**输入**：她的优点/特质
**转化**：观察欣赏点 + 选择具体类型
**输出**：她变得更有自信

**操作**：
1. 观察欣赏点：她处理冲突很冷静
2. 选择类型：行为/特质/过程/存在
3. 具体表达：「你刚才那样说话，既表达了立场又没有伤人」

**自检**：她不觉得讨好
**升级**：她开始主动分享更多

---

### 第六阶：理解系统 → 建立内在模型
**输入**：重复出现的行为模式
**转化**：收集模式 + 来源假设 + 时机验证
**输出**：能准确预测她的1个需求

**操作**：
1. 收集重复模式：每次提妈妈语速变快
2. 来源假设：可能关系紧张
3. 时机验证：「你提妈妈时好像有点激动？」
4. 行动转化：「你讨厌被催，所以我不问进度」

**自检**：能预测她的需求
**升级**：她觉得你是最懂她的人

---

### 第七阶：爱系统 → 稳定表达关爱
**输入**：她的脆弱时刻
**转化**：保持内核稳定 + 行动表达 + 脆弱陪伴
**输出**：她感到安全港湾

**操作**：
1. 保持内核稳定：不因她情绪波动而变
2. 行动表达：带她提过的零食
3. 脆弱陪伴：安静陪着，不说「别哭」
4. 尊重边界：「我就在隔壁」

**自检**：她脆弱时你能稳住
**升级**：她在你面前完全敞开

---

### 第八阶：被爱系统 → 接收与回馈循环
**输入**：她表达的爱意
**转化**：识别 + 接收 + 感受 + 回馈
**输出**：爱的流动自然循环

**操作**：
1. 识别爱意：她记得我不吃香菜
2. 接收不推：「谢谢」而不是「不用」
3. 感受自己：「我心里暖暖的」
4. 表达被爱：「你这样做，我觉得很幸福」

**自检**：能安心接收，不欠债感
**升级**：关系进入正向循环

## 八、样本库扩展（50+场景）

### 她/他的情绪样本（20+）
1. 她说「唉」→ 烦躁/累/委屈 → 被倾听
2. 她沉默 → 可能压抑/思考 → 给空间或轻问
3. 她突然客气 → 可能在防御 → 降低压力
4. 她问「在吗」→ 有话想说 → 给开口机会
5. 她说「没事」→ 可能不是 → 轻问确认
6. 她分享好消息 → 希望被认可 → 具体回应
7. 她抱怨 → 想发泄 → 先共情
8. 她道歉 → 害怕被责怪 → 先接纳
9. 她迟到 → 可能内疚 → 先共情
10. 她不回复 → 可能有情绪 → 等或轻问

### 自己的反应样本（15+）
1. 我急于给建议 → 焦虑/想帮忙 → 先倾听
2. 我沉默 → 不知道说什么 → 准备话术
3. 我反驳 → 防御/被误解 → 先确认
4. 我逃避 → 害怕冲突 → 面对但温和
5. 我讨好 → 害怕不被喜欢 → 真诚表达

### 人性规律样本（20+）
1. 人在脆弱时希望被理解
2. 人在被否定时容易防御
3. 人在被认可时更有动力
4. 人在被接纳时愿意敞开
5. 人在感到安全时敢于表达
6. 人在不确定时需要确认
7. 人在压力大时需要支持
8. 人在被误解时需要澄清
