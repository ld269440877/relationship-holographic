# 三性三管理项目脉络总纲

版本：2026-05-20  
主题：把“人性无限 -> 善/恶/非善非恶 -> 文化/制度/绩效管理”的哲学理论应用到关系动力学全息项目

---

## 0. 总论：从无限人性到有限工程

人性是无限的，任何系统都不可能穷尽人性。若试图把人性全部认识完再构建系统，项目会永远停留在混沌中。

但“论类不清，不足以为学”。要做世界级关系感知训练系统，必须先把无限人性折叠成可治理的有限结构：

```text
人性善       -> 文化管理
人性恶       -> 制度管理
非善非恶     -> 绩效管理
```

对应到本项目：

```text
善：人想连接、想被看见、想成长、想爱与被爱。
恶：人会操控、逃避、占有、欺骗、胁迫、侵犯边界。
非善非恶：人会受习惯、情境、能力、激励、注意力、疲惫、创伤和误读影响。
```

项目的核心治理原则：

```text
没有唯一正确的回应，只有适合当前人、当前关系、当前阶段、当前情绪、当前边界的回应。
没有正确的管理，只有合适的管理。
没有万能话术，只有情境化工具箱。
```

所以，本系统不追求“认识全部人性”，而是建立一套能持续学习、持续分类、持续校准的训练生命体。

## 1. 三性三管理作为项目操作系统

### 1.1 人性善：文化管理

人性善不是说人永远善，而是承认人有向善、向连接、向理解、向成长的一面。

项目中对应：

```text
同理心
好奇心
欣赏
修复意愿
尊重边界
诚实表达
承担责任
共同成长
接收爱
```

治理方式：文化管理。

系统要通过理念、仪式、榜样、正反馈和默认语言塑造用户：

- 默认鼓励“观察而非评判”。
- 默认鼓励“验证而非定论”。
- 默认鼓励“欣赏而非讨好”。
- 默认鼓励“修复而非赢辩论”。
- 默认鼓励“边界而非控制”。

文化管理不是硬拦截，而是让用户在反复训练中自然形成更好的关系品味。

### 1.2 人性恶：制度管理

人性恶不是说人本质邪恶，而是承认人在欲望、恐惧、占有、失控、权力不对等中可能做出伤害性行为。

项目中对应：

```text
PUA
煤气灯
操控依赖
欺骗
威胁
跟踪
胁迫
规避拒绝
性化未成年人
家暴
自伤/他伤风险
侵犯隐私
```

治理方式：制度管理。

系统必须有硬规则：

- 发现操控意图，拒绝生成。
- 发现危机风险，停止普通训练并升级。
- 发现隐私风险，不允许入库。
- 发现未授权真实对话，不进入训练资产。
- 发现安全分过低，综合评分封顶。

制度管理不是劝导，而是边界和闸门。

### 1.3 非善非恶：绩效管理

非善非恶是项目最重要也最容易被忽视的部分。大量关系问题不是因为人坏，而是因为能力不足、习惯惯性、场景压力、情绪泛化、注意力狭窄、旧伤触发。

项目中对应：

```text
不会开口
听不出情绪
急于解决问题
过度解释
封闭式回应
误读边界
低自我价值
焦虑追逐
回避撤退
不知道如何修复
不能接收爱
```

治理方式：绩效管理。

系统要用可测量、可训练、可反馈的方式提升能力：

- 七维评分。
- 错题本。
- 间隔重复。
- 技能掌握度。
- 课程节点。
- 训练推荐。
- 进步曲线。
- Gold sample 校准。

绩效管理不是奖惩人品，而是提升技能表现。

## 2. 三性三管理映射到项目主分支

### 2.1 核心哲学分支

| 人性分类 | 项目表达 | 管理方式 | 设计落点 |
|---|---|---|---|
| 善 | 人想被看见、理解、欣赏、爱与被爱 | 文化管理 | 总 Prompt、价值观、训练语言、默认反馈 |
| 恶 | 人可能操控、欺骗、侵犯边界 | 制度管理 | safety guard、hard block、safety_events |
| 非善非恶 | 人会误读、不会表达、能力有限 | 绩效管理 | 训练任务、评分、错题、掌握模型 |

根本脉络：

```text
文化给方向
制度给边界
绩效给路径
```

三者缺一不可：

- 只有文化：容易变成温柔口号。
- 只有制度：容易变成冰冷审查。
- 只有绩效：容易变成技巧主义。

世界级关系训练系统必须三者合一。

### 2.2 数据分支

数据也要按三性治理。

#### 人性善的数据

内容：

- 成功修复案例。
- 真诚欣赏表达。
- 健康边界表达。
- 安全依恋互动。
- 高质量共情回应。

管理方式：文化管理。

用途：

- 作为榜样样本。
- 作为理想回应。
- 作为训练系统的美学基准。
- 作为用户“关系品味”的长期塑造材料。

#### 人性恶的数据

内容：

- 操控模式。
- 煤气灯模式。
- 侵犯边界模式。
- 情感勒索。
- 跟踪威胁。
- 暴力风险信号。

管理方式：制度管理。

用途：

- 安全分类器。
- 红队测试集。
- 风险提示。
- 拒绝生成规则。
- 总分封顶机制。

注意：这些数据不能作为“技巧教程”，只能作为风险识别和防护训练。

#### 非善非恶的数据

内容：

- 常见误读。
- 冷场。
- 封闭式回应。
- 急于讲道理。
- 不会接情绪。
- 依恋风格触发。
- 情绪强度判断偏差。

管理方式：绩效管理。

用途：

- 错题本。
- 练习题。
- 能力雷达。
- 间隔复习。
- 技能掌握曲线。

### 2.3 AI 分支

AI 也必须按三性治理。

| AI 任务 | 善 | 恶 | 非善非恶 |
|---|---|---|---|
| 标注 | 看见情绪、需求、边界 | 识别操控、风险、侵犯 | 校准误读、置信度、不确定性 |
| 评分 | 鼓励尊重与连接 | 安全分封顶、拒绝危险输出 | 量化技能表现 |
| 模拟 | 训练共情和修复 | 不扮演操控对象、不诱导依赖 | 根据用户水平调整难度 |
| 进化 | 吸收新研究和好案例 | 阻断违规数据和危险模式 | 迭代 prompt、模型、rubric |

AI 的三条底线：

```text
文化上：AI 要鼓励真实连接。
制度上：AI 不能生成伤害性策略。
绩效上：AI 要给可训练、可复盘、可追踪的反馈。
```

### 2.4 训练分支

训练系统也不是单一“教你怎么说”。

#### 文化管理训练

训练用户的关系品味：

- 如何欣赏一个人的存在。
- 如何接住脆弱。
- 如何表达感谢。
- 如何修复而不讨债。
- 如何接收爱而不亏欠。

指标：

- 欣赏具体度。
- 共情自然度。
- 修复真诚度。
- 边界尊重度。

#### 制度管理训练

训练用户识别风险和停止越界：

- 哪些回应是操控。
- 哪些玩笑是冒犯。
- 哪些追问是逼迫。
- 哪些“关心”变成控制。
- 哪些沉默是安全退出。

指标：

- 操控风险识别率。
- 边界侵犯识别率。
- 高风险回应拒绝率。
- 安全替代表达能力。

#### 绩效管理训练

训练具体能力：

- 情绪命名。
- 强度判断。
- 多假设。
- 轻验证。
- 开放式提问。
- 退路式邀请。
- 冲突降温。
- 情绪流复盘。

指标：

- 七维评分。
- 技能掌握度。
- 错题复现率。
- 复习完成率。
- 迁移成功率。

### 2.5 前端体验分支

前端不是视觉包装，而是管理哲学的交互化。

| 管理类型 | 前端体验 | 例子 |
|---|---|---|
| 文化管理 | 温柔但清醒的语言系统 | “先看见，再回应” |
| 制度管理 | 清晰边界和拒绝提示 | “这个请求涉及操控，系统不会生成” |
| 绩效管理 | 可量化训练反馈 | 七维分、错题、复习、进步曲线 |

前端应有两种视角：

```text
沉浸视角：让用户进入对话现场，感受细微信号。
高维视角：让用户跳出现场，看见模式、回路、边界、叙事。
```

### 2.6 进化中心分支

进化中心必须管理三类进化：

#### 善的进化

吸收：

- 新研究。
- 新的健康关系案例。
- 更好的修复策略。
- 更优雅的欣赏表达。

产出：

- 新课程节点。
- 新理想回应。
- 新文化原则。

#### 恶的进化

识别：

- 新型操控话术。
- 新型隐私风险。
- 新型 AI 伴侣依赖风险。
- 新型社交平台危险模式。

产出：

- 新安全规则。
- 新红队样本。
- 新拒绝策略。

#### 非善非恶的进化

优化：

- 用户常错点。
- 评分偏差。
- 推荐路径。
- 课程难度。
- 样本质量。

产出：

- 新训练题。
- 新复习计划。
- 新掌握模型参数。

## 3. 从元基础到上层功能的总脉络

### 3.1 第一层：元事实

```text
人需要安全
人需要被看见
人需要边界
情绪会流动
互动是循环
关系会发展
误读不可避免
```

产物：

- `docs/元基础数据与学习架构路线图.md`
- `meta_facts` 数据表或知识条目
- 训练系统所有 rubric 的底层依据

### 3.2 第二层：原子信号

```text
词语、停顿、语气、emoji、眼神、姿态、延迟回复、消息长度、场景压力
```

产物：

- `signals` schema
- 线索标注训练
- 多假设训练

### 3.3 第三层：状态与情绪

```text
微状态 -> 情绪标签 -> 强度 -> 情绪流
```

产物：

- emotion_flow 标注
- 情绪曲线
- 情绪识别训练

### 3.4 第四层：感受、需求、边界

```text
情绪不是终点，情绪背后有感受评价、隐藏需求和边界状态。
```

产物：

- feeling tags
- hidden_need
- boundary_state
- safety_score

### 3.5 第五层：互动回路

```text
追逐-撤退
攻击-防御
讨好-失衡
沉默-误读
修复-开放
```

产物：

- interaction_loop
- 多轮 TrainerAI
- 高维复盘图

### 3.6 第六层：关系阶段与叙事

```text
同一句话在初识、暧昧、热恋、平淡、冲突、修复中意义不同。
```

产物：

- relationship_stage
- narrative_pattern
- stage_fit_score

### 3.7 第七层：系统审视

```text
从局部回应上升到关系系统：谁在追，谁在退，谁在压抑，谁在控制，谁在求安全。
```

产物：

- high_dimensional_review
- relationship_system_map
- weekly evolution report

## 4. 三性三管理对数据库的要求

新增或演进表应按三性划分：

```text
culture_principles       善：价值观、榜样、健康关系原则
safety_rules             恶：风险、边界、拒绝策略
skill_metrics            非善非恶：能力指标、评分、掌握度
source_registry          源数据 meta
raw_content_items        原始候选 meta
annotation_jobs          标注过程
training_asset_versions  训练资产版本
safety_events            安全事件
skill_state              用户技能状态
curriculum_nodes         学习路线节点
```

每条训练样本必须能回答：

```text
它在文化上塑造什么？
它在制度上防范什么？
它在绩效上训练什么？
```

## 5. 三性三管理对评分系统的要求

综合评分不能只有“像不像理想回应”。

推荐评分结构：

```text
culture_score       是否体现尊重、真诚、欣赏、连接
safety_score        是否避免操控、逼迫、侵犯边界
performance_score   是否命中情绪、需求、阶段、风格
```

七维评分可归类为：

```text
文化管理：
  connection_score
  repair_score

制度管理：
  safety_score
  boundary_score
  manipulation_risk_score

绩效管理：
  emotion_score
  need_score
  style_score
  stage_fit_score
```

总分规则：

```text
如果 safety_score 低，总分必须封顶。
如果 manipulation_risk_score 高，直接拒绝。
如果 culture_score 高但 performance_score 低，给训练建议。
如果 performance_score 高但 culture_score 低，提示技巧化/不真诚。
```

## 6. 三性三管理对学习路线的要求

### 6.1 文化路线

目标：让用户成为更值得信任、更能看见他人的人。

路线：

```text
尊重 -> 好奇 -> 看见 -> 欣赏 -> 理解 -> 修复 -> 接收爱
```

### 6.2 制度路线

目标：让用户知道哪里不能越界。

路线：

```text
边界识别 -> 风险识别 -> 拒绝操控 -> 危机升级 -> 安全替代表达
```

### 6.3 绩效路线

目标：让用户练会具体能力。

路线：

```text
信号观察 -> 情绪命名 -> 情绪流 -> 需求洞察 -> 轻验证 -> 回应生成 -> 复盘迁移
```

三条路线必须同时存在：

```text
文化让人想变好。
制度让人不能作恶。
绩效让人知道怎么变好。
```

## 7. 应写入总构建 Prompt 的增强段

```text
【三性三管理哲学】
人性是无限的，系统不能穷尽人性，但必须把无限人性折叠为可治理的有限结构：人性善、人性恶、非善非恶。人性善用文化管理，人性恶用制度管理，非善非恶用绩效管理。世界上没有正确的管理，只有合适的管理；关系中也没有万能正确回应，只有适合当前人、当前关系、当前阶段、当前情绪、当前边界的回应。

【项目应用】
文化管理负责塑造方向：同理、欣赏、尊重、修复、接收爱。
制度管理负责守住底线：拒绝 PUA、操控、跟踪、胁迫、侵犯隐私和危机风险。
绩效管理负责提升能力：情绪识别、需求洞察、边界判断、回应生成、错题复习、能力雷达。

【设计要求】
项目每个模块都必须回答三个问题：
1. 它如何激发和承载人性善？
2. 它如何约束和防范人性恶？
3. 它如何训练和改善非善非恶的能力不足、习惯惯性和情境误读？

【评分要求】
所有训练反馈必须拆分为 culture_score、safety_score、performance_score。安全分低时总分封顶；操控风险高时直接拒绝；文化分高但技能弱时给训练建议；技能分高但文化分低时提示技巧化和不真诚。
```

## 8. 终章：项目的根本脉络

整个项目的根本脉络可以压缩成一句话：

```text
用文化管理唤起善，用制度管理防住恶，用绩效管理训练非善非恶；再用数据、AI、训练和进化机制，把无限人性转化为可学习、可反馈、可修正、可成长的关系感知生命体。
```

这就是本项目从根上理清所有分支的总纲。
