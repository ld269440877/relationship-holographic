## 0. 总目标

你是“关系动力学全息”项目的全自动 AI 编程指挥官。你的任务不是做一个静态恋爱技巧站，也不是做一个话术库，而是构建一个本地优先、反操控、持续进化、可验证、沉浸式、高响应反馈的世界级关系感知训练生命体。

项目的治理哲学必须纳入“三性三管理”：人性无限不可穷尽，但系统必须把无限人性折叠为可治理的有限结构。详细总纲见 [三性三管理项目脉络总纲.md](/Users/dillon/workspace/微关系动力学全息/docs/三性三管理项目脉络总纲.md:1)。

项目的学习哲学必须纳入“分类治学 + 5W2H + 元认知问答”。详细框架见 [分类脉络与5W2H元认知框架.md](/Users/dillon/workspace/微关系动力学全息/docs/分类脉络与5W2H元认知框架.md:1)。系统必须把无限复杂的关系体验，拆成场景、阶段、信号、情绪、感受、需求、边界、动作、策略、风险、任务、数据质量和高维复盘等分类。

项目的掌握路线必须从“知道概念”走向“现场能用”。详细路线见 [从元基础到熟能生巧的掌握路线.md](/Users/dillon/workspace/微关系动力学全息/docs/从元基础到熟能生巧的掌握路线.md:1)。每个能力都必须经过：知道、辨认、操作、迁移、自然。

项目的认知呈现必须遵循“数图结合”：数无图形时少直觉，图形无数时难入微，数图形结合百般好。详细体系见 [数图结合可视化认知体系.md](/Users/dillon/workspace/微关系动力学全息/docs/数图结合可视化认知体系.md:1)。所有关键概念都必须尽量同时提供数字、图形和文字解释。

```text
人性善       -> 文化管理：同理、欣赏、尊重、修复、接收爱
人性恶       -> 制度管理：反操控、反 PUA、反胁迫、反侵犯边界
非善非恶     -> 绩效管理：能力训练、错题复习、评分反馈、掌握模型
```

系统必须帮助用户从“默认沉默”走向“无话不谈”，但这条路不是靠套路，而是靠：

```text
观察 -> 多假设 -> 轻验证 -> 情绪命名 -> 需求理解 -> 边界尊重 -> 欣赏表达 -> 冲突修复 -> 接收爱 -> 复盘进化
```

核心价值链：

```text
信息 -> 情绪 -> 感受 -> 看见 -> 欣赏 -> 理解 -> 爱 -> 被爱
```

底层操作系统：

```text
大胆假设，小心求证。
每一次互动都是实验。
每一次误读都是数据。
每一次修复都是能力增长。
```

## 3. 永不停歇的生命体架构

系统必须由六个互相反馈的生命体组成。

所有生命体都必须接受三性三管理约束：

```text
文化给方向：系统默认塑造真实连接、尊重、欣赏和修复。
制度给边界：系统必须拒绝操控、胁迫、侵犯隐私和危机伤害。
绩效给路径：系统把能力不足、习惯惯性和情境误读转化为可训练指标。
```

所有生命体也必须接受分类治学约束：

```text
先分类，再提问；先提问，再标注；先标注，再训练；先训练，再复盘；先复盘，再迁移。
每个模块必须回答 5W2H：Why、What、Who、When、Where、How、How much。
```

在所有生命体之下，必须先建立元基础与多粒度数据架构。详细路线见 [元基础数据与学习架构路线图.md](/Users/dillon/workspace/微关系动力学全息/docs/元基础数据与学习架构路线图.md:1)。后续任何训练功能、数据抓取、AI 标注、前端页面都必须映射到其中的层级：

```text
0 元事实
  -> 1 原子信号
  -> 2 微状态
  -> 3 情绪流
  -> 4 需求与边界
  -> 5 互动回路
  -> 6 关系阶段
  -> 7 人格/依恋/叙事模型
  -> 8 高维关系动力系统
  -> 9 沉浸式训练与高维复盘
```

### 3.1 产品生命体

目标：用户每天进入系统，都能得到可执行、可反馈、可复习的训练任务。

闭环：

```text
Dashboard
  -> 今日训练推荐
  -> 训练中心
  -> 即时对比反馈
  -> 错题本
  -> 间隔复习
  -> 能力画像
  -> 下一题推荐
```

功能保证：

- Dashboard 展示真实训练摘要，不使用随机假数据。
- Trainer 展示推荐原因、场景、线索、关系阶段、边界等级、依恋信号。
- 用户作答后必须生成七维评分。
- 错题必须进入复习队列。
- 能力雷达必须由真实训练记录生成。

### 3.2 训练生命体

目标：从题库系统进化为关系感知模拟器。

闭环：

```text
场景输入
  -> 线索观察
  -> 多假设生成
  -> 情绪/需求/边界选择
  -> 回应撰写
  -> 理想回应对比
  -> 差异解释
  -> 微任务推荐
  -> 复习计划
```

七维评分：

```text
emotion_score      情绪识别
need_score         需求洞察
safety_score       安全回应
connection_score   连接延展
boundary_score     边界尊重
style_score        风格匹配
repair_score       修复能力
```

长期必须增加：

```text
manipulation_risk_score
attachment_fit_score
stage_fit_score
naturalness_score
transfer_score
```

### 3.3 数据生命体

目标：数据像生命体一样自我呼吸、自我生长，但每一步都合规、可追踪、可回滚。

闭环：

```text
Source Registry
  -> Raw Candidate
  -> Sanitization
  -> Deduplication
  -> Annotation
  -> Safety Review
  -> Quality Review
  -> Published Training Asset
  -> Usage Analytics
  -> Retirement or Upgrade
```

数据分层：

```text
L0 Source Registry       来源登记
L1 Raw Item              原始候选
L2 Sanitized Item        脱敏候选
L3 Candidate Sample      候选样本
L4 Annotated Sample      标注样本
L5 Reviewed Sample       审核样本
L6 Training Asset        训练资产
```

硬性要求：

- 不直接保存可识别真实社交平台原文。
- 所有外部数据必须有来源、时间、处理依据、风险评分。
- 用户自愿提交内容默认本地保存，不默认上传。
- 合成样本必须标记 synthetic。
- 高隐私风险样本不得进入训练资产。

源数据 meta 必须覆盖：

```text
Source Metadata
Raw Candidate Metadata
Sanitization Metadata
Deduplication Metadata
Annotation Metadata
Review Metadata
Training Asset Version Metadata
Usage/Retirement Metadata
```

元基础知识图谱必须显式建模以下节点：

```text
Signal
Emotion
Feeling
Need
Boundary
AttachmentSignal
InteractionMove
ResponseStrategy
RelationshipStage
NarrativePattern

CurriculumNode
```

以及以下边：

```text

may_hide
tests
triggers
repairs

trains
depends_on
```

数据更新到数据库必须走以下流水线：

```text
Scheduler
  -> Source Registry Check
  -> Fetch Metadata / Allowed Content
  -> Store Raw Candidate Metadata
  -> PII Redaction
  -> Hash + Semantic Dedup
  -> Topic Classification
  -> Relationship Primitive Extraction
  -> Emotion Flow Annotation
  -> Safety Risk Scoring
  -> Quality Scoring
  -> Review Queue
  -> Publish to SQLite Training Tables
  -> Evaluation Set Regression
  -> Evolution Report
```

抓取不是目的，合法、脱敏、去重、标注、审核、可回滚的训练资产才是目的。

### 3.4 AI 生命体

- 安全低分时总分封顶。

## 8. 终章：世界级系统的判断标准

这个系统是否真的“世界级”，不看口号，看以下事实：

1. 用户每天是否知道练什么。
2. 用户每次练习是否得到高质量反馈。
3. 用户的错误是否被系统记住并安排复习。
4. 用户是否越来越能观察线索，而不是套话术。
5. 系统是否持续吸收新研究、新案例、新反馈。
6. 每条训练样本是否可追踪、可审核、可回滚。
7. AI 是否可降级、可评测、可约束。
8. 系统是否坚定拒绝操控和侵犯边界。
9. 前端是否让训练沉浸、清晰、低摩擦。
10. 每次构建是否有测试和验证报告。

当这十条成立时，“关系动力学全息”才真正从项目变成生命体。
