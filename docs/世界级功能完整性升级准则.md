# 世界级功能完整性升级准则

## 总原则

所有功能都必须同时满足“结构完整、内容完整、来源合规、可练可用、可审计可回滚”。数量不能替代质量，标签变化不能冒充内容变化。

## 第三方内容规则

第三方网站不默认全文搬运。默认只保存：

- 原文链接
- 标题
- 来源摘要
- 合规短摘录
- 结构化分析
- 本地原创改写与练习案例

只有在明确许可、开放协议或用户提供合法授权时，才允许保存较长正文。任何来源不明、版权不清、隐私风险高的内容进入 `quarantine` 或 `metadata_only`，不进入默认学习页面。

## 功能完整性最低标准

每个核心功能页面至少具备：

- 目标：这个页面帮助用户完成什么任务。
- 结构：页面内容按概念、案例、方法、练习、复盘或审计分区。
- 数据来源：说明数据来自数据库、用户行为、项目原创还是第三方入口。
- 质量状态：显示 published、reviewed、quarantine、draft 等状态。
- 操作闭环：至少有一个可执行动作，如加入训练、错题改写、复盘、收藏或跳转。
- 审计路径：能追溯来源、批次、质量分、变体签名或生成规则。

## 资源内容完整性最低标准

默认展示资源必须具备：

- 具体场景
- TA 原话
- 表层信号
- 深层可能
- 常见失误
- 为什么错
- 更好回应
- 边界提醒
- 练习任务
- 迁移场景
- 真实变体差异
- 来源边界

缺少以上关键字段的资源不进入默认资源海洋。

## 真实变体标准

同一主题下的变体必须至少两项不同：

- 场景不同
- TA 原话不同
- 常见失误不同
- 更好回应不同
- 练习任务不同
- 边界提醒不同
- 难度干扰条件不同
- 依恋适配策略不同

只改变标题、标签、难度数字、语气词，不算变体。

## 执行状态

已落地：

- 资源海洋默认只展示带案例蓝图的资源。
- 旧模板资源进入 `quarantine`，不删除，保留审计。
- 详情页显示来源边界、案例蓝图、完整性检查和旧版隔离提示。
- 第三方来源规则在详情页显式展示。

后续每个功能页面都按本准则逐项补齐。
