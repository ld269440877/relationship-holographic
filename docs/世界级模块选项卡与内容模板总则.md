# 世界级模块选项卡与内容模板总则

## 为什么这份总则重要

本项目不是资料堆叠，也不是话术集合。每个功能模块都必须把自己的职责、学习路径、案例证据、对话差异、情绪流动、边界出口和可执行练习讲清楚。  
这份总则的地位类似 `andrej-karpathy-skills` 对编码行为的约束：以后修订页面、扩展数据库、采集网络信息、生成资源、写知识卡，都必须先按模板思考，再落地内容。

## 总原则

1. **职责先行**：每个模块先说明它解决什么问题，不用功能名替代职责。
2. **场景落地**：每个 tab 都要落到真实场景，不能只有抽象总结。
3. **差异可见**：不同 tab 必须让用户看出学习任务、输入、输出和判断标准的差异。
4. **案例完整**：核心学习 tab 必须包含故事、对话、低质回应、更好回应、拆解、迁移练习。
5. **多维呈现**：每个模块按 5W2H、场景、情绪流、表达工具、边界、证据和行动组织。
6. **反操控**：价值降噪、问细节、现实-感受交替、情绪共振、深层情绪反馈、非语言共情都必须以安全连接为目标，不以控制对方为目标。
7. **可验证**：模板必须有机器可读字段和验证脚本，不能只靠人工感觉。

## 标准模块结构

```text
模块定位
-> 用户问题
-> 选项卡地图
-> 每个 tab 的职责
-> 内容骨架
-> 案例/故事/对话/对比
-> 5W2H
-> 场景和情绪流
-> 价值降噪与边界出口
-> 数据来源和质量门槛
-> 用户下一步行动
```

## 单个 Tab 的世界级模板

| 层级 | 必填内容 | 说明 |
|---|---|---|
| 任务层 | `jobToBeDone` | 用户点这个 tab 是为了完成什么任务 |
| 认知层 | `concept / principle / misconception` | 定义、原则、常见误解 |
| 结构层 | `fiveW2H` | Why/What/Who/When/Where/How/How much |
| 场景层 | `scenarios` | 至少 3 个真实场景 |
| 案例层 | `caseStory` | 一个完整故事，不是摘要口号 |
| 对话层 | `dialogue` | TA 原话、低质回应、更好回应 |
| 对比层 | `comparison` | 旧做法和新做法的差异 |
| 情绪层 | `emotionFlow` | 情绪从哪里来、如何变化、哪里要停 |
| 表达层 | `expressionMoves` | 价值降噪、问细节、现实-感受交替、情绪共振等 |
| 边界层 | `boundaryAndConsent` | 可拒绝、可跳过、可退出 |
| 证据层 | `evidence` | 数据字段、训练记录、来源、质量分 |
| 行动层 | `primaryAction / nextAction` | 用户下一步做什么 |

## 内容卡标准模板

```text
标题：具体场景 + 能力点 + 难度
概念：这个能力到底是什么
原则：为什么这样做
故事：一个完整小场景
TA说：对方的原话
旧回应：常见低质回应
为什么错：它怎样造成压力、误判或断联
更好回应：可直接练习的一句话或一段话
拆解：现实层 / 感受层 / 关键词 / 情绪 / 边界
情绪流：触发 -> 表层情绪 -> 深层需要 -> 降压动作 -> 可退出
5W2H：为什么、是什么、谁用、何时、哪里、如何、掌握到什么程度
迁移：换一个场景如何使用
质量门槛：具体性、可练性、边界清晰、来源合规
```

## 数据采集模板

第三方来源不能默认全文搬运。采集和扩库只允许保存：

| 字段 | 说明 |
|---|---|
| `sourceTitle` | 来源标题 |
| `sourceUrl` | 可点击链接 |
| `sourceType` | 社区、课程、播客、研究、开源、工具 |
| `shortExcerpt` | 合规短摘录 |
| `summary` | 自己写的摘要 |
| `structuredAnalysis` | 场景、情绪、边界、表达工具、风险 |
| `localOriginalRewrite` | 项目原创改写 |
| `licenseBoundary` | 许可和不可做事项 |
| `qualityScore` | 具体性、可练性、来源可信、边界清晰 |
| `templateMapping` | 映射到哪个模块、哪个 tab、哪个能力点 |

## 数据库扩展模板

任何新增数据表或字段，都要回答：

1. 它服务哪个模块和 tab？
2. 它支持哪个用户任务？
3. 它如何进入页面展示？
4. 它如何进入训练、错题、AI 伴侣或知识中枢？
5. 它如何去重、审计、回滚？
6. 它是否可能保存第三方全文或敏感私聊？
7. 它的最低质量阈值是什么？

## 各模块独特职责

### 资源海洋

职责：把内容变成可练资源，而不是资源链接仓库。  
必须呈现：完整案例、资源来源、表达工具映射、训练入口、重复治理。  
独特模板：`案例矩阵 tab`、`来源许可 tab`、`表达工具反查 tab`、`迁移练习 tab`。

### 表达工具箱

职责：帮助用户选择“怎么说”，不是背话术。  
必须呈现：工具定义、公式、适用场景、禁用动作、正反例、组合链。  
独特模板：`原子工具 tab`、`工具链 tab`、`场景检索 tab`、`安全边界 tab`。

### 知识中枢

职责：解释“为什么这样做”，让知识能指导训练。  
必须呈现：概念图谱、5W2H、知识卡、场景索引、学习路径。  
独特模板：`概念图谱 tab`、`5W2H tab`、`场景索引 tab`、`路径证据 tab`。

### 训练中心

职责：把理解转成可评分的动作。  
必须呈现：题目、作答、反馈、工具评分、推荐资源、错题入口。  
独特模板：`当前题 tab`、`七维反馈 tab`、`推荐资源 tab`、`复习计划 tab`。

### AI 伴侣

职责：在安全边界内陪练对话和状态变化。  
必须呈现：对话流、角色状态、情绪曲线、工具解释、错题记忆。  
独特模板：`对话 tab`、`状态曲线 tab`、`工具解释 tab`、`记忆错题 tab`。

### 错题本

职责：把失败回应变成下一次更好的证据。  
必须呈现：错因、旧回应、新回应、三版改写、掌握证据。  
独特模板：`待复习 tab`、`错因归类 tab`、`三版改写 tab`、`掌握证据 tab`。

### 浏览冲浪

职责：管理外部信息源，而不是把第三方内容塞进数据库。  
必须呈现：来源链接、摘要、健康状态、采集策略、许可边界。  
独特模板：`全部来源 tab`、`中文来源 tab`、`研究资料 tab`、`链接健康 tab`、`采集策略 tab`。

### 治理、分析、审计

职责：让系统质量可见、可追溯、可回滚。  
必须呈现：质量报告、发布队列、重复治理、批次回滚、运行日志。  
独特模板：`质量总览 tab`、`发布队列 tab`、`重复治理 tab`、`批次回滚 tab`、`运行日志 tab`。

## 深聊能力必须贯穿的表达模板

这些不是单独一个模块的内容，而是贯穿资源、表达、知识、训练、AI 伴侣和错题本的表达动作：

| 能力 | 页面呈现方式 | 数据字段建议 |
|---|---|---|
| 价值降噪 | 少展示自己，多看见对方 | `value_quieting_note` |
| 问细节 | 从关键词进入现实细节 | `detail_question` |
| 现实-感受交替 | 现实问一句，感受问一句 | `reality_question` / `feeling_question` |
| 情绪共振 | 先接住感受，再问内容 | `emotion_resonance` |
| 深层情绪反馈 | 说出孤独、挫败、期待等深层感受 | `deep_emotion_feedback` |
| 非语言共情 | 眼神、点头、前倾、停顿 | `nonverbal_cue` |
| 边界出口 | 可以不说、晚点说、只说一部分 | `boundary_exit` |

## 每次修订前的检查

1. 这个改动服务哪个模块职责？
2. 这个 tab 是否让职责更清楚？
3. 有没有完整案例、对话、对比和练习？
4. 有没有 5W2H 和情绪流？
5. 有没有边界出口和反操控说明？
6. 数据来源是否合规？
7. 页面是否可读、可扫、可操作？
8. 是否通过机器模板校验？
