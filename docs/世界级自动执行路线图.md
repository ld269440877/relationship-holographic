# 世界级自动执行路线图

更新日期：2026-05-21

## 执行原则

本项目后续不再按零散愿望推进，而按 `docs/tasks/module_dag.json` 自动选择下一项依赖已满足的最高优先级支柱。每一轮必须完成四件事：实现、验证、状态同步、记录下一步。

## 七大支柱

1. 数据生命体：来源登记、元数据抓取、脱敏、去重、标注、训练资产版本、进化报告。
2. 训练掌握：八阶课程图谱、掌握阶段、错题归因、间隔复习、推荐器。
3. 关系感知：信号、情绪流、需求、边界、互动回路、AI 伴侣状态轨迹。
4. 安全治理：反 PUA、反操控、反跟踪、反胁迫、危机升级、安全事件审计。
5. 前端沉浸：训练中心、AI 伴侣、八阶路径、知识图谱、进化驾驶舱、错题复盘。
6. AI 质量：Provider 适配、提示版本、结构化输出、安全降级、运行日志。
7. 工程门禁：pytest、ruff、mypy、前端 type-check/build、浏览器 smoke、数据库健康。

## 自动执行节奏

每轮自动执行：

1. 读取 `docs/tasks/module_dag.json`。
2. 运行 `python -m tools.commander run-next --dry-run` 找到下一支柱。
3. 实现该支柱最小闭环，不扩大无关范围。
4. 执行该支柱 `verification` 中的命令。
5. 若失败，先修复失败原因，再重新验证。
6. 验证通过后，更新 DAG、`docs/progress.md`、`docs/validation_report.md`。
7. 再次运行指挥官，确认下一项已切换。

## 最终世界级验收

最终不是功能堆满，而是所有支柱共同达标：

- 安全：高风险操控、胁迫、跟踪、危机场景可阻断、可解释、可审计。
- 学习：用户能从默认沉默循证推进到被爱与回馈，且每一阶有证据。
- 数据：所有训练资产来源可追踪、去重、脱敏、评分、审核、可回滚。
- 前端：核心页面不只是列表，而是数图结合的训练驾驶舱。
- AI：有版本、有日志、有降级、有结构化质量门禁。
- 工程：关键路径测试稳定，自动指挥官能持续选择下一支柱。
