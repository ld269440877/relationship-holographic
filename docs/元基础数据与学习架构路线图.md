# 元基础数据与学习架构路线图

版本：2026-05-20  
目标：把“关系动力学全息”从训练页面升级为从 0/1 元基础到高级多维认知的世界级关系感知训练生命体。

---

## 0. 核心判断

世界级关系感知系统不能直接从“话术”和“案例”开始。它必须先建立底层元基础，再让数据、知识、训练和 AI 都围绕这些元基础生长。

正确构建顺序：

```text
0 元事实
  -> 1 原子信号
  -> 2 微状态
  -> 3 情绪流
  -> 4 需求与边界
  -> 5 互动回路
  -> 6 关系阶段
  -> 7 人格/依恋/叙事模型
  -> 8 高维关系动力系统
  -> 9 沉浸式训练与高维复盘
```

系统最终必须同时做到两件事：

```text
沉浸其中：能听见一句话里的情绪、欲望、犹豫、试探、退缩。
置身其外：能从高维看见模式、周期、权力、依恋、边界、叙事和演化。
```

## 1. 0/1 底层架构：关系动力学元基础

### 1.1 0 层：不可再分的元事实

这些是系统所有数据标注、训练和评分的底座。

| 元事实 | 定义 | 系统含义 |
|---|---|---|
| 人是有身体的 | 情绪先通过身体和神经系统出现 | 训练不能只读文字，要标注语气、节奏、姿态、生理线索 |
| 人需要安全 | 没有安全感就不会开放 | 安全分必须是评分底线 |
| 人需要被看见 | 被准确感知会产生连接 | “看见”必须拆成行为、情绪、需求三层 |
| 人需要边界 | 亲密不等于融合 | 所有训练必须尊重选择权和退路 |
| 人通过故事理解自己 | 关系不是事实堆砌，而是叙事 | 复盘要记录对方和自己的故事线 |
| 情绪会流动 | 情绪有起点、高峰、转折、余波 | 样本必须标注情绪轨迹 |
| 互动是循环 | A 的表达影响 B，B 的回应反过来塑造 A | 训练不能只评价一句话，要评价回路 |
| 误读不可避免 | 猜错是数据，不是失败 | 系统要训练轻验证，而不是武断判断 |
| 关系会发展 | 同一句话在不同阶段意义不同 | 样本必须有阶段标签 |
| 权力会存在 | 关系中有主动、被动、资源、需求差异 | 安全系统要识别操控与压迫 |

### 1.2 1 层：原子信号

原子信号是最小可观察单位。

```text
词语
标点
语气
语速
停顿
重复
表情
眼神
身体朝向
距离
触碰
撤退
主动分享
转移话题
延迟回复
消息长度
emoji
时间点
场景压力
关系历史
```

每个原子信号必须支持：

```json
{
  "signal_id": "sig_pause_001",
  "modality": "text|voice|face|body|timing|context",
  "observable": "停顿 2 秒",
  "possible_meanings": ["犹豫", "压抑", "组织语言", "不想伤人"],
  "confidence": 0.0,
  "requires_validation": true
}
```

原则：一个信号不能直接推出一个结论，只能生成多个假设。

## 2. 多粒度知识架构

### 2.1 粒度阶梯

```text
Token        一个词、一个停顿、一个表情
Signal       可观察信号
State        当下心理/身体微状态
Emotion      情绪标签与强度
Feeling      意识化后的感受评价
Need         隐藏需求
Boundary     边界与选择权
Move         一次互动动作
Loop         互动循环
Scene        场景
Episode      一段关系事件
Pattern      重复模式
Stage        关系阶段
Narrative    自我/关系叙事
System       关系动力系统
```

### 2.2 从低维到高维

| 层级 | 问题 | 示例 |
|---|---|---|
| 信息层 | 发生了什么？ | 她说“没事” |
| 信号层 | 有哪些线索？ | 停顿、短句、没有 emoji |
| 情绪层 | 情绪是什么？ | 失落 6 + 防御 4 |
| 感受层 | 她如何体验这段关系？ | 被忽略、不被优先 |
| 需求层 | 她需要什么？ | 被重视、被解释、被安抚 |
| 边界层 | 哪些地方不能推进？ | 不能逼问、不能要求立刻好起来 |
| 回路层 | 两人如何互相触发？ | 她撤退 -> 你追问 -> 她更撤退 |
| 阶段层 | 关系发展到哪里？ | 暧昧后期/冲突修复 |
| 叙事层 | 她在讲什么故事？ | “我总是不被认真对待” |
| 系统层 | 关系动力是什么？ | 焦虑追逐 + 回避撤退 |

### 2.3 关键张力维度

世界级系统必须显式建模关系中的张力：

```text
靠近 <-> 自主
安全 <-> 新鲜
表达 <-> 保留
确定 <-> 探索
照顾 <-> 控制
幽默 <-> 冒犯
欣赏 <-> 讨好
关心 <-> 入侵
解释 <-> 防御
修复 <-> 讨债
```

每个训练题都应标注至少一个张力维度。

## 4. 数据抓取、更新数据库与进化流水线

### 4.1 合规优先的数据来源分级

| 等级 | 来源 | 入库策略 |
|---|---|---|
| A | 权威公开研究、专家博客、公开播客说明页 | 存元数据、摘要、链接、派生原则 |
| B | 公开论文和开放教材 | 存结构化摘要、概念、引用 |
| C | 用户本地自愿提交 | 本地保存，默认不上传，允许删除 |
| D | AI 合成训练样本 | 标记 synthetic，经过安全与质量审核 |
| E | 社交平台公开内容 | 默认只抽象模式，不保存可识别原文 |

抓取不是目的。目标是“可训练的抽象模式”。

### 4.2 永续更新流水线

```text
Scheduler
  -> Source Registry Check
  -> Fetch Metadata / Allowed Content
  -> Store Raw Candidate Metadata
  -> PII Redaction
  -> Hash + Semantic Dedup
  -> Topic Classification
  -> Relationship Primitive Extraction
  -> Emotion Flow Annotation
  -> Safety Risk Scoring
  -> Quality Scoring
  -> Review Queue
  -> Publish to SQLite Training Tables
  -> Evaluation Set Regression
  -> Evolution Report
```

### 4.3 数据库新增建议

在现有表基础上新增：

```sql
-- 来源登记
CREATE TABLE source_registry (
    id INTEGER PRIMARY KEY,
    source_uuid TEXT UNIQUE NOT NULL,
    name TEXT NOT NULL,
    source_type TEXT NOT NULL,
    url TEXT,
    trust_score REAL DEFAULT 0.8,
    update_frequency TEXT DEFAULT 'weekly',
    allowed_use_json TEXT,
    disallowed_use_json TEXT,
    active BOOLEAN DEFAULT 1,
    last_checked_at DATETIME,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 原始候选元数据
CREATE TABLE raw_content_items (
    id INTEGER PRIMARY KEY,
    raw_uuid TEXT UNIQUE NOT NULL,
    source_id INTEGER,
    title TEXT,
    url TEXT,
    content_hash TEXT,
    raw_storage_policy TEXT,
    privacy_risk REAL DEFAULT 0,
    copyright_risk REAL DEFAULT 0,
    consent_status TEXT,
    processing_status TEXT DEFAULT 'raw',
    collected_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY(source_id) REFERENCES source_registry(id)
);

-- 脱敏与去重日志
CREATE TABLE content_processing_logs (
    id INTEGER PRIMARY KEY,
    raw_item_id INTEGER,
    step TEXT NOT NULL,
    status TEXT NOT NULL,
    report_json TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY(raw_item_id) REFERENCES raw_content_items(id)
);

-- 标注任务
CREATE TABLE annotation_jobs (
    id INTEGER PRIMARY KEY,
    target_type TEXT NOT NULL,
    target_id INTEGER NOT NULL,
    annotator_type TEXT NOT NULL,
    annotator_version TEXT,
    schema_version TEXT,
    result_json TEXT,
    confidence REAL DEFAULT 0,
    status TEXT DEFAULT 'draft',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 训练资产版本
CREATE TABLE training_asset_versions (
    id INTEGER PRIMARY KEY,
    asset_type TEXT NOT NULL,
    asset_id INTEGER NOT NULL,
    version TEXT NOT NULL,
    source_trace_json TEXT,
    quality_json TEXT,
    review_status TEXT DEFAULT 'draft',
    published_at DATETIME,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## 8. 沉浸与高维审视的双视角设计

### 8.1 沉浸视角

让用户进入现场：

- 对方说了什么。
- 当时在哪里。
- 语气如何。
- 停顿在哪里。
- 你身体有什么反应。
- 你第一反应想说什么。
- 如果你这样说，对方下一秒可能怎么感受。

### 8.2 高维视角

让用户跳出现场：

- 这是什么互动回路？
- 谁在追，谁在退？
- 哪个边界被试探？
- 哪个需求没被看见？
- 哪个旧叙事被触发？
- 哪个回应会让系统升温？
- 哪个回应能让系统降温？

### 8.3 UI 表达

训练页面应有两个模式：

```text
沉浸模式：对话、线索、即时反应、角色反馈
高维模式：时间轴、情绪曲线、互动回路图、需求/边界图谱
```

### 9.1 第一阶段：元基础与数据骨架

```text
进化中心能展示数据流水线状态
```

目标：让训练从“回应评分”变成“感知训练”。

```text
用户能完成一套：线索 -> 多假设 -> 情绪流 -> 回应 -> 对比 -> 复习
能力雷达由 skill_state 生成
错题推荐与课程节点关联
```

### 9.3 第三阶段：进化生命体

目标：让系统持续吸收新知识，但不违规抓取。

任务：

- Source Registry 管理页面/API。
- 权威来源定期检查。
- 用户本地提交入口。
- AI 合成候选样本生成。

验收：

```text
每周能生成候选条目

发布样本能进入训练推荐
```

### 9.4 第四阶段：沉浸式模拟器

目标：从题库到关系动力模拟器。

## 10. 总构建 Prompt 增强段

把以下内容追加到总构建 Prompt：

```text
【元基础要求】
不要从话术和页面开始。必须先建立 0/1 底层架构：元事实、原子信号、微状态、情绪流、感受、需求、边界、互动回路、关系阶段、叙事模式和系统视角。

【数据抓取与更新要求】
系统必须建立从 Source Registry 到 Published Training Asset 的完整流水线。抓取不是目的，合法、脱敏、去重、标注、审核、可回滚的训练资产才是目的。公开社交平台内容默认只抽象互动模式，不保存可识别原文。

【学习路线要求】
课程必须从 Signal 级别开始，逐步进入 Emotion、Feeling、Need、Boundary、Loop、Narrative、System。每个阶段都有可观察指标、训练任务、评分标准、错题复习和晋级条件。

【双视角要求】
系统既要提供沉浸式现场体验，也要提供高维审视视角。沉浸模式训练即时感知，高维模式训练模式识别和关系系统理解。

【多粒度要求】
每条样本必须支持从 token/signal 到 scene/episode/pattern/system 的多粒度标注。每次训练必须明确当前训练的是哪一层粒度。
```
