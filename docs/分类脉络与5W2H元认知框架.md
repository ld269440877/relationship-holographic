# 分类脉络与 5W2H 元认知框架

版本：2026-05-20  
目标：通过分类、5W2H、问答框架，把关系动力学项目从“无限复杂的人性”整理成可学习、可训练、可掌握的清晰脉络。

---

## 0. 总论：分类不是简化人性，而是建立入口

人性无限，不可能被完全认识。但学习必须先分类。分类不是把人性变小，而是给无限复杂建立可进入、可观察、可训练的入口。

本项目的分类目标：

```text
混沌体验
  -> 可观察分类
  -> 可提问分类
  -> 可标注分类
  -> 可训练分类
  -> 可复盘分类
  -> 可迁移分类
  -> 高维理解
```

用户最终要获得的不是“背分类表”，而是：

```text
元感知：我正在感知什么？
元感受：我正在感受什么？
元认知：我如何理解我的理解？
元判断：我根据什么判断？
元行动：我此刻适合用什么工具？
```

## 1. 项目总分类树

整个系统按 12 个主类组织：

```text
1. 场景分类
2. 关系阶段分类
3. 信息与信号分类
4. 情绪与感受分类
5. 需求与边界分类
6. 依恋与人格倾向分类
7. 互动动作分类
8. 回应策略分类
9. 风险与安全分类
10. 学习任务分类
11. 数据来源与质量分类
12. 高维复盘分类
```

每个主类都必须回答：

```text
What：这是什么？
Why：为什么重要？
Who：涉及谁？
When：什么时候出现？
Where：在哪种场景出现？
How：如何识别/训练/回应？
How much：程度、强度、风险、优先级是多少？
```

## 2. 场景分类：Where + When

场景是理解一切关系信号的容器。同一句话离开场景就会失真。

### 2.1 一级场景

```text
初识
破冰
日常闲聊
暧昧试探
约会前
约会中
约会后
热恋亲密
平淡期维护
冲突爆发
冷战撤退
道歉修复
边界协商
异地沟通
重大压力
分离/告别
复合/重建
```

### 2.2 场景 5W2H

| 维度 | 问题 | 标注字段 |
|---|---|---|
| Why | 这个场景的关系任务是什么？ | `scene_goal` |
| What | 发生了什么事实？ | `scene_facts` |
| Who | 谁主动，谁回应？ | `actor_roles` |
| When | 关系处于什么时间点？ | `timing` |
| Where | 线上、线下、公共、私密？ | `channel_context` |
| How | 对话如何展开？ | `interaction_sequence` |
| How much | 情绪强度/风险等级多高？ | `scene_intensity` |

### 2.3 场景训练问题

```text
这个场景的真正任务是连接、试探、修复、确认，还是退出？
如果换一个场景，同一句话意义会不会改变？
对方是在表达事实，还是在发出连接请求？
这个场景最不能做的事是什么？
```

## 3. 关系阶段分类：When

关系阶段决定同一句话的含义和边界。

```text
陌生
初识
熟悉
暧昧
确认关系
热恋
稳定
平淡
压力期
冲突期
修复期
重建期
结束期
```

阶段分类要解决：

```text
什么可以说？
什么不该推进？
什么回应显得自然？
什么回应显得越界？
什么行为是正常探索？
什么行为是危险控制？
```

阶段评分：

```text
stage_fit_score：回应是否适合当前关系阶段。
boundary_fit_score：推进程度是否尊重当前边界。
```

## 4. 信息与信号分类：What

信息是事实，信号是事实里露出的关系意义。

### 4.1 信息分类

```text
事实信息：今天加班、下雨、几点见
偏好信息：喜欢什么、不喜欢什么
状态信息：累、忙、烦、开心
关系信息：想你、没事、随便、你看着办
边界信息：不想、不要、下次吧、我需要空间
元信息：回复速度、语气、字数、表情、停顿
```

### 4.2 信号分类

```text
连接请求
认可请求
安抚请求
空间请求
边界试探
撤退信号
修复信号
攻击信号
防御信号
失望信号
邀请信号
结束信号
```

### 4.3 信号问答

```text
我看见的是事实，还是我脑中的解释？
这个信号至少有哪 3 种可能含义？
我有没有把一个信号过度确定化？
这个信号需要回应，还是需要验证？
```

## 5. 情绪与感受分类：What + How Much

情绪是身体和关系系统的即时波动；感受是情绪进入意识后的关系评价。

### 5.1 情绪分类

```text
喜：满足、愉悦、期待、雀跃、兴奋
怒：不满、烦躁、恼火、愤怒、委屈性愤怒
哀：失落、难过、心酸、沮丧、绝望
惧：紧张、不安、焦虑、恐惧、恐慌
爱：好感、依恋、温柔、心动、珍惜
惊：意外、惊讶、震动、错愕
羞：不好意思、尴尬、羞耻、被看穿
混合：纠结、酸楚、忐忑、释然、又爱又怕
```

### 5.2 感受分类

```text
被看见 / 被忽略
被理解 / 被误解
被尊重 / 被冒犯
被重视 / 被敷衍
被接纳 / 被评判
安全 / 不安全
自由 / 被控制
亲近 / 被推开
有价值 / 没价值
```

### 5.3 情绪流分类

```text
起点：最初情绪
触发：哪句话/哪个行为改变了情绪
高峰：强度最高点
转折：情绪开始变化的位置
余波：对后续关系的影响
```

训练问题：

```text
对方现在的主情绪是什么？
混合情绪里哪一个更需要先被接住？
情绪强度是 3、6 还是 9？
这个情绪背后的关系评价是什么？
```

## 6. 需求与边界分类：Why + How Much

需求解释“为什么有情绪”，边界解释“哪里不能越过”。

### 6.1 需求分类

```text
被理解
被重视
被安抚
被确认
被欣赏
被选择
被尊重
被陪伴
被信任
有空间
有自主
有确定性
有新鲜感
有公平感
```

### 6.2 边界分类

```text
时间边界：现在不想聊
空间边界：需要一个人待会
身体边界：不想被触碰
情绪边界：不要逼我立刻好起来
隐私边界：不想说细节
关系边界：还没到那个阶段
承诺边界：不想给确定答案
价值边界：这个玩笑我不接受
```

### 6.3 边界等级

```text
1 轻微偏好
2 暗示
3 试探
4 犹豫
5 模糊拒绝
6 明确拒绝
7 情绪化拒绝
8 撤退
9 阻断
10 危机/安全风险
```

训练问题：

```text
这是需求信号还是边界信号？
我应该靠近、停住、退后，还是轻验证？
我有没有把对方的边界误读成“考验”？
```

## 7. 依恋与人格倾向分类：Who

依恋分类不是给人贴标签，而是帮助选择合适回应。

```text
安全型：直接表达需求，能协商
焦虑型：需要确认，害怕失去
回避型：需要空间，害怕被吞没
恐惧回避型：想靠近又怕受伤
未知型：信息不足，不做结论
```

5W2H：

```text
Who：对方当前更像哪种依恋激活？
Why：她/他害怕的是失去、被控制，还是被否定？
How：回应是给确认、给空间，还是给稳定存在？
How much：确认给多少不会变成纵容？空间给多少不会变成消失？
```

## 8. 互动动作分类：How

互动不是一句话，而是一连串动作。

```text
打开：分享、提问、自我暴露
靠近：邀请、确认、表达想念
试探：模糊表达、半开玩笑、观察反应
撤退：短句、转移、延迟、冷淡
攻击：指责、讽刺、翻旧账
防御：解释、反击、受害者姿态
修复：道歉、承认、理解、补偿
协商：提出选择、给退路、共同决定
结束：明确停止、告别、拉开距离
```

训练问题：

```text
对方这一步是在靠近、撤退、试探还是修复？
我的下一步会让回路升温还是降温？
我是在回应对方，还是在回应我的恐惧？
```

## 9. 回应策略分类：How

回应策略不是话术，而是工具。

```text
共情反射
情绪命名
轻验证
开放提问
退路式邀请
具体欣赏
边界表达
冲突降温
负责式道歉
修复协商
幽默缓冲
沉默陪伴
延迟回应

```

每个策略必须标注：

```json
{
  "strategy": "轻验证",
  "best_for": ["模糊情绪", "边界试探", "不确定信号"],
  "bad_for": ["明确拒绝后继续推进", "危机场景"],
  "culture_value": "尊重和好奇",
  "safety_rule": "不强行定义对方",
  "performance_metric": "假设数量和验证质量"
}
```

## 10. 风险与安全分类：How Much

风险分类决定是否继续训练、降级、拒绝或升级。

```text
低风险：表达笨拙、冷场、误读
中风险：边界模糊、轻度施压、情绪激烈
高风险：操控、威胁、跟踪、煤气灯、胁迫
危机风险：自伤、他伤、家暴、性侵、未成年人风险
```

系统动作：

```text
低风险 -> 训练建议
中风险 -> 安全提示 + 替代表达
高风险 -> hard block + 安全解释
危机风险 -> 停止普通训练 + 升级建议
```

## 11. 学习任务分类：How

训练不是一种题型，而是一组递进任务。

```text
观察题：找信号
分类题：信号属于哪类
情绪题：命名情绪和强度
情绪流题：画出情绪变化
需求题：提出多个隐藏需求假设
边界题：识别能否推进
回应题：写出回应
对比题：比较用户回应与理想回应
修复题：把错误回应改成修复回应
复盘题：从高维看互动回路
迁移题：把同一原则迁移到新场景
```

课程路线：

```text
元觉察
  -> 信息识别
  -> 信号观察
  -> 情绪命名
  -> 情绪流
  -> 感受翻译
  -> 需求洞察
  -> 边界判断
  -> 回应生成
  -> 回路建模
  -> 叙事理解
  -> 系统审视
```

## 13. 高维复盘分类：Why + System

高维复盘让用户置身其外。

```text
事实层：发生了什么？
信号层：有哪些线索？
情绪层：情绪如何流动？
需求层：哪些需求没被看见？
边界层：哪里被推进或越过？
回路层：两个人如何互相触发？
叙事层：谁在讲什么旧故事？
系统层：这个关系动力如何自我重复？
行动层：下一次最小改变是什么？
```

高维问答：

```text
我把什么当成了事实，其实只是解释？
我最先感受到的身体信号是什么？
我是在保护自己，还是在靠近对方？
这次互动属于哪个旧模式？
如果我是旁观者，我会看见什么？
下一次只改一个动作，改什么？
```

## 15. 应写入总构建 Prompt 的增强段

```text
【分类治学要求】
人性无限，不能穷尽；但学习必须分类。系统必须用场景、阶段、信息、信号、情绪、感受、需求、边界、依恋、互动动作、回应策略、风险、学习任务、数据质量和高维复盘等分类，把无限复杂的关系体验转化为可观察、可提问、可标注、可训练、可复盘、可迁移的结构。

【5W2H 要求】
项目每个模块都必须能回答：Why 为什么重要，What 是什么，Who 涉及谁，When 何时出现，Where 在什么场景，How 如何识别/训练/回应，How much 强度/风险/优先级是多少。

【元认知要求】
系统必须帮助用户形成元感知、元感受、元认知、元判断、元行动：我正在感知什么，我正在感受什么，我如何理解我的理解，我根据什么判断，此刻适合用什么工具。

【双层掌握要求】
训练既要沉浸其中，识别一句话里的情绪、欲望、犹豫、试探、退缩；也要置身其外，从高维看见场景、阶段、模式、边界、叙事、回路和系统演化。
```
