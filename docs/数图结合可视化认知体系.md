# 数图结合可视化认知体系

版本：2026-05-20  
主题：把“数无图形时少直觉，图形无数时难入微，数图形结合百般好”植入关系动力学全息项目。

---

## 0. 总原则

关系动力学很复杂。如果只有文字，容易抽象；只有数字，缺少直觉；只有图形，又难以细分、比较、追踪。

本项目必须采用：

```text
数字入微
图形成像
数图结合
一眼看懂
层层钻深
```

对应关系：

```text
数字：强度、频率、比例、趋势、得分、风险、掌握度
图形：结构、流动、关系、阶段、回路、路径、对比
文字：解释、假设、建议、复盘、意义
```

三者合一：

```text
数字告诉你“多少”
图形告诉你“长什么样”
文字告诉你“意味着什么”
```

## 1. 从元基础到上层建筑的数图体系

| 层级 | 数字 | 图形 | 用户直觉 |
|---|---|---|---|
| 元事实 | 安全分、边界分、连接分 | 三角基座图 | 关系先要稳 |
| 原子信号 | 信号数量、置信度 | 信号散点/高亮文本 | 线索在哪里 |
| 情绪 | 强度 1-10 | 情绪温度计 | 情绪多强 |
| 情绪流 | 起点/高峰/转折分值 | 情绪曲线 | 情绪如何变化 |
| 感受 | 被看见/被忽略得分 | 感受象限图 | 关系体验在哪里 |
| 需求 | 需求紧急度 | 需求雷达 | 最想被满足什么 |
| 边界 | 边界等级 1-10 | 红黄绿边界带 | 能不能推进 |
| 互动动作 | 动作次数/方向 | 箭头序列图 | 谁在靠近谁在撤退 |
| 回路 | 循环强度 | 互动回路图 | 模式如何重复 |
| 阶段 | 阶段适配分 | 阶段路径图 | 现在到哪一步 |
| 策略 | 策略适配分 | 工具箱矩阵 | 该用哪把工具 |
| 风险 | 风险等级 | 风险仪表盘 | 该训练、提醒还是阻断 |
| 掌握 | 掌握度 0-100 | 熟练度阶梯 | 我练到哪了 |
| 进化 | 新增/淘汰/审核数 | 生命体指标面板 | 系统有没有变聪明 |

## 2. 必备图形组件

### 2.1 情绪温度计

用途：显示单个情绪强度。

```text
1-2  微弱
3-4  轻微
5-6  明显
7-8  强烈
9-10 淹没
```

训练问题：

```text
这个情绪是 3、6 还是 9？
如果是 8，现在还适合讲道理吗？
```

### 2.2 情绪流曲线

用途：显示一次互动中情绪如何变化。

```text
强度
10 |                         *
 8 |              *        *
 6 |     *     *
 4 |  *
 2 |
 0 +--------------------------------
     起点   触发   高峰   转折   余波
```

必须标注：

```text
emotion
intensity
trigger
turning_point
repair_or_escalation
```

### 2.3 信号高亮图

用途：在原话中高亮不同信号。

```text
“没事，你忙吧。”
  ^^   ^^^
  |    撤退/失望/给台阶
  防御性最小化
```

颜色规则：

```text
蓝色：事实信息
绿色：连接信号
黄色：模糊/待验证
橙色：边界信号
红色：风险信号
```

### 2.4 需求雷达

用途：显示隐藏需求强度。

```text
被理解     80
被重视     70
被安抚     45
有空间     60
有确定性   55
```

训练问题：

```text
最强需求是哪一个？
这个需求能不能直接满足？
是否和边界冲突？
```

### 2.5 边界红黄绿带

用途：判断能否推进。

```text
绿色 1-3：可轻推进
黄色 4-6：先验证
橙色 7-8：停止推进，接情绪
红色 9-10：退出/安全升级
```

### 2.6 互动回路图

用途：显示双方如何互相触发。

```mermaid
flowchart LR
    A["她短句撤退"] --> B["我焦虑追问"]
    B --> C["她更防御"]
    C --> D["我解释更多"]
    D --> A
```

训练目标：

```text
找出一个能打断循环的最小动作。
```

### 2.7 场景-阶段地图

用途：把回应放进关系阶段。

```text
陌生 -> 初识 -> 熟悉 -> 暧昧 -> 确认 -> 热恋 -> 稳定 -> 冲突 -> 修复
```

指标：

```text
stage_fit_score
推进风险
自然度
```

### 2.8 工具箱矩阵

用途：根据场景选择工具。

| 场景 | 首选工具 | 禁用工具 | 风险 |
|---|---|---|---|
| 模糊情绪 | 轻验证 | 直接下结论 | 中 |
| 明确拒绝 | 尊重边界 | 继续推进 | 高 |
| 冲突爆发 | 降温/负责 | 讲道理 | 高 |
| 低落倾诉 | 共情陪伴 | 立刻解决 | 中 |

### 2.9 能力雷达

用途：显示用户能力结构。

```text
情绪识别
需求洞察
安全回应
连接延展
边界尊重
风格匹配
修复能力
```

升级建议：

```text
增加 culture_score
增加 safety_score
增加 performance_score
增加 stage_fit_score
增加 manipulation_risk_score
```

### 2.10 进化生命体面板

用途：显示系统自我生长。

```text
来源数
候选条目
脱敏通过率
去重率
标注置信度
审核通过率
发布样本数
拒绝样本数
安全事件数
本周新增原则
```

## 5. 数图结合的掌握路线

### 5.1 从数字入微

用户先学会：

```text
情绪强度不是“有/没有”，而是 1-10。
边界不是“能/不能”，而是 1-10。
需求不是“要/不要”，而是紧急度和冲突度。
风险不是“危险/安全”，而是等级。
```

### 5.2 从图形生直觉

用户再学会：

```text
情绪是一条曲线。
关系是一个回路。
边界是一条带。
需求是一个雷达。
能力是一张地图。
进化是一个漏斗。
```

### 5.3 从数图结合到熟能生巧

最终用户看到一个场景时，脑中自然浮现：

```text
这是什么场景？
现在是什么阶段？
情绪强度大概几分？
边界在哪个颜色？
需求雷达最高的是哪项？
互动回路正在升温还是降温？
我该用哪把工具？
```

## 6. 应写入总构建 Prompt 的增强段

```text
【数图结合要求】
数无图形时少直觉，图形无数时难入微，数图形结合百般好。系统所有关键概念都必须同时提供数字、图形和文字解释：数字负责强度、频率、风险、趋势和掌握度；图形负责结构、流动、回路、阶段和路径；文字负责意义、假设、建议和复盘。

【训练图解要求】
每个训练题必须尽量提供信号高亮、情绪温度计、情绪流曲线、需求雷达、边界红黄绿带、互动回路图和回应差异高亮。用户要一眼看懂关系动力，也能继续钻到细粒度数字。

【前端要求】
Dashboard、Trainer、Mistakes、Evolution、Knowledge 都必须以图解为核心，不做纯文字堆砌。图形不是装饰，而是认知工具。
```

---

## 7. 总结

关系动力学要真正简单易懂，必须做到：

```text
用数字入微
用图形成像
用文字释义
用训练内化
```

这样，复杂的人性和关系动力才不会停留在玄学，而会变成用户看得见、算得清、练得会、用得出的能力。
