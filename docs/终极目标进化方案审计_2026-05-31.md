## 三、分阶段进化路线图

### 🟢 第一阶段：稳定性筑基（1-2周）

> **目标**：让系统能"稳定跑起来"，解决致命短板

| 步骤 | 任务 | 验收标准 | 评级 |
|:--:|------|------|:--:|
| 1.1 | **修复 AI Provider 400 问题** | AI成功率 ≥ 75% | 🔴 P0 |
| 1.2 | **向量召回修复** | Top10 recall ≥ 80% | 🔴 P0 |
| 1.3 | 固化每日全链路健康检查 | regression-audit 每日自动通过 | 🟡 P1 |
| 1.4 | start.sh 一键启动可靠性 | 冷启动成功率 100% | 🟡 P1 |
| 1.5 | 知识中枢扩容 | entries 28→100+，每条绑定案例+工具 | 🔴 P0 |

**1.1 详细方案（AI Provider）：**
- 用 `POST /api/analytics/ai-provider-probe`（需设置 `DEEPSEEK_LIVE_PROBE_ENABLED=true`）
- 确认账号区域（中国 vs 海外）、模型授权（deepseek-v4-pro）、额度状态
- 如果 DeepSeek 不可靠，评估备选 Provider（OpenRouter/Anthropic/本地模型）
- 把 Provider 诊断集成到 start.sh 启动健康检查

**1.2 详细方案（向量召回）：**
- 重定义评测集（情绪流/边界同意/冲突修复/暧昧张力/长期连接/开放提问/NVC/镜子技术）
- 统一检索摘要字段（跨 resource_library/knowledge_entries/interaction_samples/expression_tools）
- 混合检索：BM25 + coverage_axis + tool_id + vector rerank
- 搜索结果增加"为什么推荐"解释
