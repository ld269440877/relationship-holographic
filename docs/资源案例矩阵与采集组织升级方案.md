# 资源案例矩阵与采集组织升级方案

## 核心原则

资源不再以“标签变化”冒充变体。合格资源必须包含案例骨架：场景、关系阶段、触发事件、TA 原话、表层信号、深层可能、常见失误、为什么错、更好回应、回应拆解、边界提醒、练习任务、迁移场景和真实变体差异。

## 最小内容单元

当前最小单元为：

```text
项目主线 × 场景 × 资源类型 × 难度 × 依恋适配
```

覆盖维度：

- 项目主线：微关系信号、情绪流动、边界与同意、暧昧张力、冲突修复、长期连接、幽默互动、错题改写。
- 场景：初识、暧昧、热恋、冲突、修复、长期、异地、复联、分歧、亲密推进、公开场合、压力支持。
- 资源类型：具体故事卡、三步训练卡、张力回应卡、边界短句卡、低压幽默卡。
- 难度：D1 照结构、D2 迁移、D3 复杂改写。
- 依恋适配：安全型、焦虑型、回避型、通用。

每个最小单元最低记录阈值：2 条。

## 数据库落地

新增资源字段：

- `case_blueprint_json`：案例蓝图 JSON。
- `variant_signature`：真实变体签名。
- `content_unit`：最小内容单元。
- `coverage_axis`：项目主线覆盖轴。
- `variant_family`：变体家族。
- `case_completeness_score`：案例结构完整度。

新增覆盖表：

- `resource_coverage_requirements`：每个最小单元的最低记录数。
- `resource_coverage_audit`：每次生成后的覆盖审计。

## 采集组织流程

```text
content_sources
-> raw_content_items
-> extracted_scenes
-> case_blueprints
-> resource_library
-> practice_tasks
-> review_queue
```

第三方来源默认只做元数据、链接、短导读和本地原创训练转化；只有明确许可的来源才允许保存正文。所有外部内容进入资源海洋前必须转成项目原创案例蓝图，不直接复制受版权保护的全文。

## 真实变体验收

同一主题下，任意两张卡至少两项不同：

- 场景不同。
- TA 原话不同。
- 常见失误不同。
- 更好回应不同。
- 边界提醒不同。
- 练习任务不同。
- 难度干扰条件不同。
- 依恋策略不同。

不满足的资源进入 `quarantine` 或 `needs_enrichment`，不进入默认资源海洋。
