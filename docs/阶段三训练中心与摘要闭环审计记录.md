# 阶段三：训练中心与摘要闭环审计记录

时间：2026-05-20

## 目标

继续移除静态展示和随机统计，把训练中心升级为真实刻意练习界面，并补充训练摘要 API，为 Dashboard/Daily 后续完全真实化提供后端基础。

## 已完成

1. 后端 `backend/api/training.py`
   - 新增 `GET /api/training/summary/today`
   - 新增 `GET /api/training/summary/week`
   - 输出今日训练次数、平均分、未掌握错题、完成状态、推荐建议。
   - 输出最近 7 天训练次数、活跃天数、平均分、连续训练提示。

2. 前端 `frontend/src/pages/Trainer.vue`
   - 重写为真实训练中心。
   - 使用 `store.fetchNextTraining()` 获取推荐题。
   - 展示推荐原因、场景、TA 原话、行为、阶段、难度、依恋信号、边界等级。
   - 支持柔和/张力/幽默三种回应风格。
   - 提交后展示综合分。
   - 展示七维评分：情绪识别、需求洞察、安全回应、连接延展、边界尊重、风格匹配、修复能力。
   - 展示错题入库提示。
   - 展示下一步推荐。
   - 侧栏展示真实能力雷达。

3. 前端 API/Store
   - `frontend/src/utils/api.ts` 新增 TrainingTodaySummary / TrainingWeekSummary。
   - `trainingApi.getTodaySummary()` / `trainingApi.getWeekSummary()`。
   - `frontend/src/stores/training.ts` 新增 `todaySummary`、`weekSummary`、`fetchSummaries()`。

4. 本轮审计强化
   - `backend/api/training.py` 将 `/api/training/compare` 升级为混合评分入口：规则评分为稳定基线，DeepSeek v4 结构化评分为深度增强，安全护栏或未配置 API Key 时自动降级。
   - `CompareResponse` 新增 `scoring_source` 与 `ai_feedback`，前端可明确展示“规则评分 / AI 深评 / 降级”来源。
   - `frontend/src/pages/Trainer.vue` 增加评分来源提示、继续下一题推荐、错题闭环入口。
   - `frontend/src/pages/Dashboard.vue` 接入 `fetchSummaries()`、`fetchNextTraining()`，移除本地随机统计，今日训练/连续打卡/已学样本/推荐说明均来自真实 API/Store。
   - 新增 `backend/database/import_json_content.py`，支持把 `场景样本库完整版.json`、话术库、段子库、故事库、游戏库迁入 SQLite，且通过 `json:*` UUID 保证可重复执行不重复导入。
   - `tests/test_training_flow.py` 增加 summary smoke、混合评分降级断言、JSON 导入 smoke。

5. 数据库与 HTML/知识结构分离强化
   - 新增 `backend/models/knowledge.py`：`KnowledgeSection`、`KnowledgeEntry`、`ContentImportBatch`、`ContentImportIssue`。
   - `backend/database/connection.py` 已纳入 knowledge 模型建表，知识内容进入 SQLite 单一数据源体系。
   - 新增 `backend/database/import_knowledge_content.py`，支持 `知识库.json` 与旧 Markdown 手册结构化导入；HTML 明确降级为 legacy source，不再作为主数据源。
   - 新增 `backend/api/knowledge.py`，提供 `/api/knowledge/sections`、`/api/knowledge/entries`、`/api/knowledge/entries/{id}`、`/api/knowledge/imports/latest`。
   - `backend/main.py` 已注册 Knowledge API。
   - 前端新增 `frontend/src/pages/Knowledge.vue` 知识中枢页面，所有知识展示从 API 读取，保留 `/manual` 仅作为旧 HTML 手册入口。
   - `frontend/src/router/index.ts` 新增 `/knowledge` 路由，`frontend/src/App.vue` 新增“知识中枢 DB”导航入口。
   - `tests/test_knowledge_api.py` 覆盖知识导入、API 查询、详情读取与幂等导入。
   - `tests/test_training_flow.py` 增强 JSON 内容导入幂等断言。

6. 训练闭环小型质量修复
   - `CompareResponse.dimension_scores` 改为 `Field(default_factory=dict)`，移除可变默认值风险。

   - `tests/test_knowledge_api.py` 增加结构化目录优先读取测试。

8. Daily 每日训练真实化增强
   - `frontend/src/pages/Daily.vue` 移除本地假数据：`streak = 7`、`rank = 42`、本地 `calories`。

## 当前质量判断

训练闭环已经贯穿：

```text
Trainer/Daily 前端
  → /api/training/next
  → 用户回应
  → /api/training/compare
  → TrainingAttempt
  → MistakeLog
  → Radar
  → Mistakes 复习
```

这标志着系统从“展示型训练页”进入“真实学习闭环产品”的第一阶段。

## 剩余风险

1. Daily 已接入 `fetchNextTraining()`、真实情绪标签解析、summary/due reviews/mistakes 同步；剩余风险是情绪识别任务仍未写入后端 `TrainingAttempt(mode='emotion')`。
2. DeepSeek v4 已接入 compare 混合评分，但真实效果依赖 `DEEPSEEK_API_KEY`、模型可用性和结构化 JSON 稳定性。
3. JSON 迁移脚本已具备，但尚未默认在启动流程中自动执行，避免误导入大量内容；建议先人工审计后执行。
4. TrainerAI 尚未接入 AI Orchestrator。
5. 七维评分在未配置 AI 时仍走规则派生；AI 可用时才使用结构化 `dimension_scores` 增强。
6. 知识导入 dry-run 当前已降至 `0 issue`，但仍需要真实导入后的人工抽样审计，确认内容排版、标题、摘要与标签质量。
7. `/manual` 仍保留 legacy HTML 托管入口，但已经不再作为新增知识中枢的主数据源。
