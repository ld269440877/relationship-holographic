# 阶段二：前端闭环与进化中心审计记录

时间：2026-05-20

## 完成目标

本阶段将第一轮后端真实训练闭环接入前端体验层，减少静态假数据，建立“训练-错题-进化中心”的产品入口。

## 已完成

1. `frontend/src/utils/api.ts`
   - 新增 `/api/training/next` 类型与方法。
   - 新增 `/api/training/reviews/due` 与复习提交方法。
   - 扩展 `ComparisonResult`、`RadarData`、`MistakeItem`。
   - 新增 Evolution API 类型与 `evolutionApi`。

2. `frontend/src/stores/training.ts`
   - 新增 `nextItem`、`dueReviews` 状态。
   - 新增 `fetchNextTraining()`、`fetchDueReviews()`、`submitReview()`。
   - 提交复习后自动刷新错题、到期复习和能力雷达。

3. `frontend/src/pages/Mistakes.vue`
   - 从硬编码错题改为真实 API 数据。
   - 展示未掌握错题、今日到期、累计答对、累计答错。
   - 支持“已掌握/继续复习”，接入间隔重复后端。

4. `frontend/src/pages/Daily.vue`
   - 抽题从随机样本改为 `/api/training/next`。
   - 情绪识别不再随机判定，改为解析样本 `emotion_tags_json`。
   - 回应训练继续写入后端训练记录和错题本。

5. `frontend/src/pages/Evolution.vue`
   - 新增系统进化中心页面。
   - 展示单一数据源原则、进化条目、质量分和标签。

6. `frontend/src/router/index.ts` 与 `frontend/src/App.vue`
   - 新增 `/evolution` 路由。
   - 侧边栏新增“进化中心 Live”。

7. 测试
   - 新增 `tests/conftest.py` 修复 pytest import path。
   - 新增 `tests/test_training_flow.py`。
   - 新增 `tests/test_evolution_api.py`。

## 验证结果

- `npm run build`：通过。
- `pytest tests/test_training_flow.py tests/test_evolution_api.py -q`：2 passed。
- `npm run type-check`：失败，原因是当前环境 Node.js v25.9.0 与旧版 `vue-tsc@1.8.27` 对 TypeScript 内部结构兼容性问题，不是业务代码编译错误。Vite build 已通过。

## 剩余风险

1. `Daily.vue` 仍有部分打卡、热量、排名、streak 为本地状态，后续应接入后端 summary。
2. `Trainer.vue` 仍使用旧的随机样本逻辑，应改为 `/api/training/next` 并展示七维评分。
3. Evolution 目前只有 API 与前端展示，尚未接入 Cronjob/DeepSeek 自动标注。
4. DeepSeek 骨架已建，但未接入 compare 深度评分。
5. JSON 内容尚未迁移到 SQLite。

## 下一阶段建议

1. 重构 `Trainer.vue`：统一使用 `fetchNextTraining()`，展示七维评分、错题入库提示、下一题推荐。
2. 新增 `backend/database/import_json_content.py`，分批导入 JSON。
3. 接入 DeepSeek v4 到 `compare`：规则引擎作为 fallback，AI 负责深度解释。
4. 新增 `training/summary/today` 与 `training/summary/week`，替代前端随机统计。
5. 提升测试覆盖到 API 路由级别。
