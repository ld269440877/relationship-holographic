## 1. 核心目标梳理

### 1.1 项目定位

项目定位为：本地优先、反操控、持续进化、可审计的关系动力学感知训练系统。

它训练的不是“怎么控制对方”，而是：

- 看见微关系信号。
- 命名情绪、感受和关系状态。
- 理解需求、边界、节奏和同意。
- 用低压力、可拒绝、可退出的方式表达。
- 练习幽默互动、暧昧张力、冲突修复、长期连接、爱与被爱。
- 通过训练记录、错题、复盘和能力画像形成长期进步轨迹。

### 1.2 最高边界

项目必须持续守住：

- 不生成 PUA、操控、胁迫、煤气灯、跟踪、绕过拒绝、侵犯隐私的话术。
- 不把性别刻板印象伪装成“底层逻辑”。
- 不把第三方网站全文搬运进数据库。
- 不用无效改写、换标签、换难度来假装“变体丰富”。
- 不让用户只看到抽象原则，而看不到具体场景、完整对话、差异分析和迁移练习。

### 1.3 核心学习闭环

```text
场景案例
  -> 用户作答
  -> 低质回应 / 更好回应 / 多视角回应对比
  -> 结构化反馈
  -> 错因归因
  -> 错题本与间隔复习
  -> 能力雷达
  -> 下一题推荐
```

### 1.4 内容进化闭环

```text
来源登记
  -> 原始候选
  -> 合规摘要 / 短摘录 / 本地原创改写
  -> 去重
  -> 模板补全
  -> 审核
  -> 发布
  -> 质量评估
  -> 回滚 / 隔离 / 再生成
```

### 4.2 当前主要问题

1. AI 伴侣的可用性不稳定  
   前端显示 Provider 已配置，但近期真实统计成功率只有 41.7%。这会直接影响 `/trainer-ai` 的核心体验。

2. 向量召回不足  
   资源、知识、表达工具和训练样本已经很多，但召回只有 25%，用户很可能搜索不到最合适的内容。

3. 调度健康为 0  
   `commander_sync_state_hourly` 和 `regression_audit_daily` stale，说明“自动化自主决策完整实现”还没有闭环。

4. 资源内容质量存在历史债  
   用户多次指出完整对话、场景故事、更好回应、工具案例存在重复、抽象、语境不匹配。数据库已经有治理脚本和模板，但仍需要持续批量校验。

5. 知识中枢条目偏少  
   `knowledge_entries=28`，对“关系动力学全息”这个目标来说不够。应扩展术语、机制、谱系、误区、案例、工具映射和练习。

6. 表达工具箱还不够“可举一反三”  
   工具数 63 已够，但每个工具需要标准微课程：定义、适用/禁用场景、真实对话、低质/高质对比、5W2H、情绪流、边界出口、迁移练习。

7. 数据字段与概念层存在不一致  
   文档和接口里有“使命轴/能力轴”的概念，资源表实际字段是 `coverage_axis`，不存在 `mission_axis` 和 `scene` 直接列。后续批量治理要避免脚本误读。

8. “世界级”文档很多，执行入口需要继续收敛  
   统一索引已明确 L0-L4，但仓库里仍存在大量阶段性方案文档。新开发必须以 `requirements_final.md`、`high_level_design.md` 和模板规范为准。

9. 性别/关系内容需要去刻板化  
   外部材料中容易出现“女人底层逻辑”“慕强/恋恶/受虐”等泛化标签。项目应改写为更专业的关系需求、依恋风格、情绪调节、边界同意和个体差异框架，避免强化偏见。

## 5. 建设性方案

### 5.1 内容质量治理方案

建立“案例卡最小合格标准”，所有资源卡、表达工具卡和知识练习卡必须至少具备：

| 字段 | 要求 |
|---|---|
| 场景故事 | 具体到人物、关系阶段、触发事件、关系风险。 |
| TA 原始信号 | 有真实语气的原话，不是抽象总结。 |
| 前一轮上下文 | 解释为什么 TA 会这么说。 |
| 低质量回应 | 至少 2 个不同误区：说教、审问、急解决、越界、讨好、逃避。 |
| 更好回应 | 至少 2-3 个视角：温和承接、开放提问、边界出口、轻张力或修复版。 |
| 差异分析 | 说明好在哪里，不只给答案。 |
| 5W2H | 谁、何时、何地、发生什么、为什么、怎么说、说到什么程度。 |
| 情绪流 | 表露前、表露中、回应后可能的情绪变化。 |
| 表达工具 | 映射到具体工具，如镜子技术、现实-感受交替、NVC 三步。 |
| 边界与安全 | 可拒绝、可停下、不施压、不替对方做决定。 |
| 举一反三 | 至少 3 个不同场景迁移：暧昧、社交、职场/长期关系等。 |

建议新增或强化的质量指标：

- `case_completeness_score >= 85` 才能进入默认主列表。
- `better_response_context_match_score >= 80` 才能展示为“更好回应”。
- `semantic_duplicate_score < threshold` 才能作为变体。
- 每个表达工具至少绑定 5 个真实场景案例。
- 每个知识概念至少绑定 3 个训练样本、3 个资源、1 个表达工具。

### 5.2 AI 可靠性方案

P0 目标：把 AI 成功率从 41.7% 拉回 75% 以上。

执行路径：

1. Provider 诊断  
   用 dry-run readiness 确认本地 request shape；在受控环境里再开启 live probe，确认账号区域、模型授权、服务侧参数约束。

2. 响应结构化修复  
   对 `raw_text`、缺少 `reply`、score schema 不完整建立回归样例。所有 AI 任务必须有 schema normalization。

3. fallback 体验升级  
   当 Provider 不可用时，不只显示失败，而是返回本地模板化练习：场景复述、情绪识别、开放提问、边界出口、三步改写。

4. 安全阻断替代路径  
   安全阻断后必须给用户“可练习”的替代方案，而不是只拒绝。

### 5.3 检索与推荐方案

P0/P1 目标：把向量 top10 recall 从 25% 提升到 80%。

执行路径：

- 重新定义评测集：覆盖情绪流、边界同意、冲突修复、暧昧张力、长期连接、开放提问、非暴力沟通、镜子技术等。
- 对 `resource_library`、`knowledge_entries`、`interaction_samples`、`expression_tools` 统一生成检索摘要字段。
- 做混合检索：关键词 BM25 / SQLite LIKE + coverage_axis / tool_id / scene filter + vector rerank。
- 搜索结果展示“为什么推荐”：匹配了哪个场景、情绪、工具、边界、训练目标。

### 5.4 学习路径方案

以用户当前短板“连接延展”为第一批强化路径：

```text
开放式提问
  -> 问细节
  -> 现实-感受交替
  -> 捕捉关键词
  -> 区分事实与情绪
  -> 镜子技术
  -> 深层情绪反馈
  -> 边界出口
  -> 迁移到暧昧/社交/职场/长期关系
```

每个技能建立：

- 术语定义。
- 禁用误区。
- 三档难度。
- 真实对话。
- 低质/高质对比。
- 练习题。
- 错题归因。
- 与表达工具箱的映射。
- 与资源海洋的案例联动。

### 5.5 自动化治理方案

要实现“全自动化自主决策”，不能只靠脚本存在，必须有状态机和门禁：

```text
每日回归审计
```

## 9. 总体建议

下一步不要继续无限新增“宏大模块”，而是围绕四条主线做收敛：

1. 稳定运行：AI、调度、构建、审计每日稳定。
2. 内容可信：每张卡都有具体场景、完整对话、好坏对比和可迁移练习。
3. 学习有效：围绕能力短板推荐路径，真正让用户从知道到会用。
4. 治理可持续：每次采集、补全、改写、发布、隔离都有质量门槛和审计证据。

“任何事情的发展都没有奇迹，只有轨迹。”  
这个项目的世界级路线不是一次性堆满所有内容，而是让每一次采集、每一次修复、每一次训练、每一次审计都留下轨迹，并且这条轨迹能持续把系统推向更具体、更安全、更有效。
