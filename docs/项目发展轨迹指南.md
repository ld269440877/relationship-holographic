# 项目发展轨迹指南

> 任何事情的发展都没有奇迹，只有轨迹。

这句话是本项目后续数据扩展、页面升级和训练质量治理的共同准则：不要期待一次性“变成世界级”，而是让每一轮新增都留下来源、结构、训练样本、反馈和审计证据。

## 轨迹闭环

1. 来源登记：记录名称、链接、可信度、更新频率、允许使用和禁止使用。
2. 原始锚点：只保存标题、URL、摘要、短摘录、结构化观察和 hash。
3. 结构化分析：拆出主题、场景、情绪流、边界、表达工具、5W2H 和风险。
4. 本地原创训练卡：生成场景故事、完整对话、低质量回应、更好回应、多角度变体和练习任务。
5. 练习反馈：用用户表现更新掌握模型、错题归因和资源推荐。
6. 审计报告：检查重复、低质、来源健康、字段完整度和版权边界。
7. 下一轮扩展：只根据缺口补数据，避免盲目堆量。

## 5W2H

- Why：提升关系感知、表达选择和可迁移练习，不是堆素材。
- What：每条最小记录都要有来源、结构、场景、对话、边界和练习。
- Who：资源海洋、表达工具箱、知识中枢、训练中心、AI 伴侣和错题本共同使用。
- When：当某类场景、标签、表达目的、来源组或表达工具低于阈值时补齐。
- Where：SQLite 是主真源，文档是设计索引，页面是学习入口。
- How：先登记来源，再原创转化，再审核发布，再用反馈迭代。
- How much：追求每轮可验证新增、可回滚、可复盘。

## 质量红线

- 不把不同标签套在同一段话上伪装成变体。
- 不默认搬运第三方全文。
- 不让卡片只有概念，没有场景故事、对话和可练任务。
- 不让页面只有数据堆，没有检索、分组、目录和详情。
- 不让新增数据绕过来源、版权、隐私、去重和发布状态。

## 信息源采集模板

每个高质量来源至少记录：

- 来源名称、URL、来源组、可信度、更新频率。
- 可用字段：标题、链接、短摘录、摘要、结构化分析、本地原创改写。
- 禁用字段：第三方全文、私聊原文、未授权批量复制、没有边界出口的案例。
- 主题：如边界同意、情绪流动、冲突修复、长期连接、幽默互动。
- 场景：如初识、暧昧、热恋、冲突、修复、平淡、承诺、分歧、异地、复联。
- 转化结果：资源锚点、完整案例卡、知识条目、表达工具映射、训练任务。

## 验收

每轮扩展至少验证：

- 数据库新增记录数量和幂等性。
- `/api/resources/sources` 能返回新增来源。
- `/surf` 能展示新增来源组、主题和场景。
- 知识中枢能检索到本指南。
- 测试、类型检查和构建通过。
