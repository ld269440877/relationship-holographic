## 统一功能原则

1. **反操控优先**：所有表达、深聊、暧昧、修复和训练功能都以安全连接为目标，不以控制对方为目标。
2. **职责先行**：资源海洋、表达工具箱、知识中枢、训练中心、AI 伴侣、错题本、浏览冲浪、治理审计必须各自承担清晰职责。
3. **案例完整**：核心学习内容不能只给标题或抽象总结，必须尽量包含故事、TA 原话、低质回应、更好回应、拆解、迁移练习。
4. **多维组织**：内容按场景、阶段、信号、情绪、感受、需求、边界、表达工具、风险、训练任务和 5W2H 组织。
5. **来源合规**：第三方网站不默认全文搬运，只保存链接、标题、摘要、短摘录、结构化分析和本地原创改写。
6. **数据可审计**：导入、生成、发布、隔离、回滚和链接健康都要有批次、状态、质量分和日志。
7. **页面可操作**：多记录页要有搜索、筛选、目录、详情、空状态和下一步动作；tab 不是装饰，而是工作区。
8. **可验证交付**：每次功能变更必须至少通过一种相关门禁：pytest、模板校验、类型检查、构建、smoke 或回归审计。

## 新增或修改功能的标准流程

```text
1. 对齐 L0：确认目标不违背反操控、安全边界和项目主线。
2. 对齐 L1：确认功能属于哪个页面需求、训练闭环、进化闭环或安全闭环。
3. 对齐 L2：确认 API、数据库和页面职责边界。
4. 对齐 L3：按模块 tab 模板补齐案例、对话、5W2H、情绪流、边界和数据契约。
5. 执行最小实现：只改完成目标所需的文件。
6. 验证：运行对应测试、模板校验、类型检查或浏览器 smoke。
7. 归档 L4：更新 progress、validation report 或任务状态。
```

## 后续维护规则

- 不再新增孤立的“世界级方案”文档作为执行入口；新想法先映射到 L1-L3。
- 可以保留研究、审计和阶段记录，但必须在文件中说明它们是背景材料还是执行契约。
- 任何新增模块 tab，必须同步 `docs/tasks/module_tab_templates.json` 并通过模板校验。
- 任何新增采集脚本，必须遵守来源合规原则，不保存第三方全文或敏感私聊原文。
- 任何新增资源生成逻辑，必须提供具体案例变体，不能只换标签、难度或语气。
