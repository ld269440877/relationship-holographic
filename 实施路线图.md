# 🧭 微关系动力学全息系统 - 实施路线图

## 一、项目愿景

> **"形无数时难入微，数无形时少直觉"**
> 
> 从"不知道自己不知道"到"知道自己知道并能做到"
> 
> 构建全球最专业、最懂用户的**动态进化型两性关系动力学系统**

## 二、项目定位

### 2.1 核心功能
- **个人成长系统**：从认知到行为，从知道到做到
- **关系导航系统**：从陌生到相知到相爱
- **情感流动系统**：从表面到深度到极致
- **人性探索系统**：从自我到他人到众生

### 2.2 核心价值
- 对比驱动学习（不是记住知识，而是形成直觉）
- 定时自动更新（静态数据库 → 动态生命体）
- 持续进化机制（每天进步一点点）

## 五、核心系统详解

### 5.1 对比引擎（核心）

**对比类型**：

| 类型 | 说明 | 应用场景 |
|------|------|----------|
| 情绪标尺对比 | 你的判断 vs 专家判断 | 情绪识别训练 |
| 回应风格对比 | 你的回应 vs 三种风格 | 回应优化训练 |
| 时间轴对比 | 今天 vs 昨天 vs 上周 | 进步追踪 |
| 横向对比 | 你 vs 平均 vs 优秀 | 定位水平 |

**对比流程**：
```
1. 用户输入/选择
      ↓
2. 生成对比数据（用户 vs 理想）
      ↓
3. 差异分析（情绪词/需求/温度/问句/留白）
      ↓
4. 可视化呈现（高亮+图表）
      ↓
5. 改进建议生成
      ↓
6. 存入成长数据库
```

### 5.2 定时更新系统

**更新架构**：
```
┌─────────────────────────────────────────────────────────┐
│  定时触发层（Cron Scheduler）                             │
├─────────────────────────────────────────────────────────┤
│  每日任务（凌晨2:00）                                   │
│  ├── 采集新样本（知乎/B站/小红书热帖）                    │
│  ├── 检测外部资源是否更新                                │
│  └── 清理过期链接                                        │
│  每周任务（周日凌晨3:00）                                │
│  ├── 全量去重（智能去重算法）                            │
│  ├── 情绪标签补全                                        │
│  └── 生成数据报告                                        │
│  每月任务（1号凌晨4:00）                                │
│  ├── 错题本统计                                          │
│  ├── 用户能力雷达图更新                                  │
│  └── 新维度/新资源推荐                                   │
└─────────────────────────────────────────────────────────┘
```

### 5.3 八阶成长路径

| 阶 | 名称 | 目标 | 输入→转化→输出 |
|---|------|------|----------------|
| 0 | 默认沉默 | 启动侦探模式 | 没话题→注意力转向外部→注意到3个细节 |
| 1 | 信息交换 | 交换基本信息 | 不知道问什么→环境切入+开放问题→5+事实 |
| 2 | 情绪识别 | 识别情绪标签 | 她说的话→找情绪词+多假设→准确率>50% |
| 3 | 感受共鸣 | 建立情感连接 | 她的情绪→接住+命名+陪伴→她说"对，就是这样" |
| 4 | 看见系统 | 照出未意识状态 | 微表情/微行为→捕捉+假设+验证→她眼睛睁大 |
| 5 | 欣赏系统 | 照亮对方优点 | 她的优点→观察+选择类型→她更有自信 |
| 6 | 理解系统 | 建立内在模型 | 重复模式→收集+假设+验证→能预测需求 |
| 7 | 爱系统 | 稳定表达关爱 | 她的脆弱→内核稳定+陪伴→她感到安全 |
| 8 | 被爱系统 | 接收与回馈 | 她的爱意→识别+接收+回馈→爱的循环 |

## 八、技术栈

### 8.1 前端
- **框架**：Vue 3 + Vite
- **UI**：Tailwind CSS
- **状态管理**：Pinia
- **图表**：ECharts
- **存储**：IndexedDB（本地）+ SQLite

### 8.2 后端/调度
- **运行环境**：Node.js 18+
- **调度器**：whenly（轻量级持久化）
- **数据库**：SQLite（本地）+ PostgreSQL（生产）
- **爬虫**：Playwright（动态页面）

### 8.3 核心库
- **diff**：文本差异对比
- **date-fns**：日期处理
- **idb**：IndexedDB封装
